# 阿里云邮件推送（DirectMail）配置指南

本文档记录 Meego Lite 接入阿里云邮件推送（DirectMail）作为验证码/通知邮件发送通道的完整配置流程。

适用场景：解决 QQ 等公共 SMTP 服务发件被企业邮箱（如 `lmdgame.com`）静默拒投、收件人收不到验证码的问题。

---

## 一、为什么用阿里云邮件推送

| 维度 | QQ / 163 SMTP | 阿里云邮件推送（DirectMail） |
|---|---|---|
| 发件信誉 | 一般，企业邮箱常拦 | 高信誉 IP 池，送达率高 |
| SPF / DKIM / DMARC | 自己配，麻烦 | 平台引导一键配置 |
| 退信可观测性 | 黑盒，看不到拒收原因 | 控制台「邮件追踪」可看每封状态 |
| 适合场景 | 个人/小测试 | 验证码、通知、事务邮件 |
| 价格 | 免费 | 200 封/天免费，超出 2 元/1000 封 |

---

## 二、前置条件

| 条件 | 说明 |
|---|---|
| 阿里云账号 | 已实名认证（个人/企业均可） |
| 域名 | 一个真实可用的域名（如 `xxx.com`），已实名认证完成、状态为「正常」 |
| ICP 备案 | **不需要**（仅用于发邮件，不解析到国内服务器作网站时无需备案） |

> 💡 仅发邮件不强制要求备案；只有当域名解析到中国内地服务器对外提供网站访问时才需要 ICP 备案。

---

## 三、整体流程概览

```
[1] 清理误建发信域名
       ↓
[2] 邮件推送 → 新建发信域名 mail.xxx.com
       ↓
[3] 阿里云 DNS 解析 → 添加 4 条记录（TXT + MX + SPF + DKIM）
       ↓
[4] 等待 DNS 生效 → 邮件推送点「验证」
       ↓
[5] 创建发信地址 noreply@mail.xxx.com → 设置 SMTP 密码
       ↓
[6] 修改项目 .env → 重启服务
       ↓
[7] 触发验证码发送 → 控制台「邮件追踪」确认送达
```

预计总耗时：**15–45 分钟**（其中 DNS 生效需要 5–30 分钟）。

---

## 四、详细操作步骤

### 第 1 步：清理之前的错误配置

打开 https://dm.console.aliyun.com → 左侧「**发信域名**」 → 删除曾经误建的 `xxx.aliyunmail.com` 等无效域名。

### 第 2 步：在邮件推送新建发信域名

1. 仍在「发信域名」页 → 点「**新建域名**」
2. 填写：**`mail.your-domain.com`**（推荐使用三级子域名）
   - 不要直接用主域 `your-domain.com`，避免与未来网站邮件冲突
3. 提交后状态为「**待验证**」
4. 该域名行 → 点「**配置**」或「**DNS 设置**」 → 查看待添加的 4 条 DNS 记录

阿里云会给出类似如下记录（**具体值以控制台显示为准**）：

| # | 类型 | 主机记录 | 记录值 | 备注 |
|---|---|---|---|---|
| 1 | TXT | `aliyundm.mail` | `aliyun-domain-verification=xxxxxxxx` | 所有权验证 |
| 2 | MX | `mail` | `mxdm.aliyun.com`（优先级 10） | 退信接收 |
| 3 | TXT | `mail` | `v=spf1 include:spf1.dm.aliyuncs.com -all` | SPF 反垃圾 |
| 4 | TXT | `dkim._domainkey.mail` | `v=DKIM1;k=rsa;p=MIGfMA0GCSqGSIb...`（很长） | DKIM 签名 |

### 第 3 步：去 DNS 解析控制台添加 4 条记录

打开 https://dns.console.aliyun.com → 找到 `your-domain.com` → 「**解析设置**」 → 「**添加记录**」。

按上表逐条添加。各字段填写要点：

- **主机记录**：只填子域前缀，不要带主域。如 `mail`，不要写成 `mail.your-domain.com`
- **MX 优先级**：填 `10`
- **DKIM 记录值**：完整复制粘贴，**不要截断、不要换行、不要带引号**
- **TTL**：默认 600 秒即可

添加完成后，DNS 解析列表中应有 4 条新记录，状态均为「**正常**」。

### 第 4 步：等待 DNS 生效并验证

#### 4.1 等待 DNS 生效

阿里云域名 + 阿里云 DNS：通常 1–5 分钟生效，最多 30 分钟。

#### 4.2（可选）本地验证 DNS 是否生效

```bash
dig TXT mail.your-domain.com
dig MX mail.your-domain.com
dig TXT dkim._domainkey.mail.your-domain.com
```

能查到对应记录即生效。

#### 4.3 回邮件推送点「验证」

回到 https://dm.console.aliyun.com → 发信域名 → 找到 `mail.your-domain.com` → 「**验证**」。状态变为「**已验证**」。

> ⚠️ 验证失败常见原因：
> - DNS 还未生效，等几分钟再试
> - 主机记录填错（错把 `mail` 写成 `mail.your-domain.com`）
> - DKIM 记录值被截断或带了多余引号

### 第 5 步：创建发信地址

#### 5.1 新建发信地址

邮件推送控制台 → **发信地址** → 「**新建发信地址**」：

| 字段 | 填法 |
|---|---|
| 发信域名 | 选 `mail.your-domain.com` |
| 账号 | `noreply` |
| 发信地址类型 | **触发邮件**（验证码必选这个） |
| 回信地址 | 填一个能收信的常用邮箱（用于接收退信通知） |
| 邮件类型 | 触发短信（重要业务通知） |

最终发信地址：**`noreply@mail.your-domain.com`**

#### 5.2 设置 SMTP 密码

发信地址列表 → 「**设置 SMTP 密码**」：

- 自己设一个强密码（建议 16+ 位，含大小写+数字）
- ⚠️ **不是阿里云登录密码**，是该发信地址专用的 SMTP 密码
- 立即复制保存（关闭窗口后看不到原文）

### 第 6 步：修改项目 `.env`

在项目根目录的 `.env` / `.env.local` 中，把原来的 SMTP 配置（如 QQ 邮箱）替换为：

```dotenv
# 阿里云邮件推送 SMTP 配置
SMTP_HOST=smtpdm.aliyun.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=noreply@mail.your-domain.com
SMTP_PASS=你刚才设置的SMTP密码
SMTP_FROM=noreply@mail.your-domain.com
```

支持的端口：

| 端口 | 加密方式 | `SMTP_SECURE` |
|---|---|---|
| **465** | SSL | `true`（推荐） |
| 80 / 25 | 明文 | `false`（部分云厂商封 25 端口） |
| 587 | STARTTLS | `false` |

注意事项：

- 确认 `.gitignore` 里有 `.env`，不要把密钥提交到 git
- 部署在服务器上的项目，**记得同步修改服务器上的 `.env`**

### 第 7 步：重启服务并测试

#### 7.1 重启 Meego Lite

本地开发：

```bash
npm run dev
```

服务器（PM2）：

```bash
pm2 restart meego-lite
```

#### 7.2 触发验证码发送

通过前端注册/登录页面输入测试邮箱，点击「发送验证码」。

#### 7.3 检查后端日志

期望日志：

```
INFO [send-code] 收到验证码请求 email=test@example.com purpose=register
INFO [notification] 邮件发送成功 to=test@example.com
  messageId=<xxxxx@smtpdm.aliyun.com>      ← messageId 来自 smtpdm 即说明走阿里云
  accepted=1 rejected=0
```

#### 7.4 收件方查收

- 收件箱
- 垃圾邮件箱
- 企业邮箱管理员隔离区（部分企业邮箱会拦截到此处）

### 第 8 步：在阿里云控制台查看送达情况

阿里云邮件推送提供详细的送达可观测性：

- **控制台 → 发送数据 → 数据统计**：每天发送/送达/退信总量
- **控制台 → 发送数据 → 邮件追踪**：每封邮件具体状态
  - 送达 ✅
  - 拒收（退信，会显示具体错误码）
  - 打开（如果收件方打开了邮件）

若某封显示拒收，可看到具体原因，便于针对性处理（如对方域名黑名单、SPF 校验失败等）。

## 六、常见问题

### Q1：为什么发信域名要用三级子域 `mail.xxx.com`，不直接用 `xxx.com`？

主域留作其他用途（网站、未来其他邮箱），三级域专用于邮件推送，避免 DNS 记录互相干扰。

### Q2：DNS 添加完了，验证为什么一直失败？

最常见原因排查顺序：

1. DNS 还未生效（用 `dig` 或 https://www.whatsmydns.net 查全球生效情况）
2. 主机记录填错（多/少了主域名后缀）
3. DKIM 记录值被截断 / 多了空格 / 多了引号
4. 同一主机记录有多个 TXT 记录冲突

### Q3：阿里云提示"今日额度已用完"怎么办？

- 默认日额度 2000 封，可随信誉等级提升
- 控制台可申请"提升额度"
- 或购买资源包

### Q4：发信被收件方退信怎么办？

在「邮件追踪」中查看具体退信原因：

- `5.1.1 User unknown` → 收件邮箱不存在
- `5.7.1 spam policy / blocked` → 对方反垃圾策略拦截，需联系对方 IT 加白名单
- `5.7.x SPF check failed` → SPF 记录配置不正确，重新检查 DNS

### Q5：可以同时配置多个发信域名吗？

可以。一个账号可以配多个域名/地址，按业务隔离。例如：

- `noreply@mail.xxx.com` → 系统通知
- `support@mail.xxx.com` → 客服回复

### Q6：阿里云邮件推送会审核内容吗？

会。验证码、通知、密码重置等事务邮件正常使用无审核问题。营销邮件需另行申请权限。
