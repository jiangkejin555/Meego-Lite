# 通知配置指南

Meego Lite 支持四种通知渠道：**站内通知（in_app）**、**邮件（email）**、**飞书（feishu）**、**企业微信（wecom）**。

- 站内通知开箱即用，无需任何配置；
- 邮件走 SMTP，需要配置一次发件账号（管理员级，全局共用）；
- 飞书 / 企业微信走"群机器人 Webhook"，每个用户在自己的设置页里粘贴各自的 Webhook 即可。

通知核心实现位于 `src/lib/notification.ts`。

## 一、邮件通知（QQ 邮箱 SMTP）

邮件渠道使用 `nodemailer` 通过 SMTP 协议发送，配置写在项目根目录的 `.env`，**全局共用一个发件账号**，所有需要发邮件的通知都从这个账号发出。

### 1.1 申请 QQ 邮箱授权码

1. 浏览器登录 [QQ 邮箱](https://mail.qq.com)（用要作为发件人的那个 QQ 账号）
2. 顶部 **设置 → 账号**
3. 找到 **POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV 服务**
4. 点 **IMAP/SMTP 服务** 旁的 **"开启"**
5. 按提示用绑定的手机号发送指定短信完成验证
6. 验证成功后页面会显示一串 **16 位授权码**（形如 `abcdefghijklmnop`），**只显示一次，立即复制保存**
7. 弄丢了可以回到同一位置点 **"生成授权码"** 再生成一份

> 授权码不是 QQ 登录密码，只用于 SMTP/IMAP 协议，泄漏可在后台关闭重新生成。

### 1.2 写入 `.env`

打开项目根目录的 `.env`，按以下格式填写：

```bash
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your_account@qq.com
SMTP_PASS=刚才那串16位授权码
SMTP_FROM="Meego Lite <your_account@qq.com>"
```

⚠️ **注意事项**

- `SMTP_FROM` 中的发件邮箱**必须等于 `SMTP_USER`**，否则 QQ 会以 `mail from address must be same as authorization user` 拒收；显示名（前面那段）随意。
- 不要使用 `${SMTP_USER}` 这种变量插值写法，dotenv 默认不展开，请直接写完整地址。
- `.gitignore` 已包含 `.env*`，凭据不会被提交。
- 如果 465 端口被网络限制，可改成 `SMTP_PORT=587` 同时 `SMTP_SECURE=false`（STARTTLS 模式）。

### 1.3 配额与限制

- QQ 个人邮箱日发送量约 **50–500 封**，超量会被临时限流
- 单封最多约 50 个收件人
- 团队大量发送请改用**腾讯企业邮箱**或其它专业邮件服务

### 1.4 验证

1. 重启 `bun run dev`
2. 在「设置 → 通知」里把目标用户的 `notifyEmail` 打开
3. 触发任意一次任务分配，或调用 `POST /api/notifications/check-deadlines`
4. 在通知中心查看记录，`status=sent` 即成功；若 `failed`，错误原因写在 `error` 字段

## 二、飞书通知（自定义群机器人）

飞书走的是**群机器人 Webhook**，无需创建开发者应用，零成本。每个用户在自己的设置页粘贴各自的 webhook URL，通知会被推送到对应的群。

> 想要"私聊推送"？飞书允许**只有自己一个人的群**，把机器人加到这种单人群里，效果等同于私发。

### 2.1 获取 Webhook URL

1. 在飞书里找到一个目标群聊，没有就：左上角 **+ → 发起群聊** 新建一个（可以只选自己）
2. 进入群聊 → 点右上角群设置（齿轮 / "…"）
3. 找到 **群机器人 → 添加机器人**
4. 选 **自定义机器人**（Custom Bot）
5. 填写机器人名称（如 `Meego Lite`）和可选描述、头像
6. 进入**安全设置**，三选一（至少选一种）：
   - **自定义关键词**（推荐）：填一两个关键词，例如 `任务`，发出的消息必须包含其中之一才会成功
   - **IP 白名单**：桌面应用不推荐（IP 会变）
   - **签名校验**：最安全，但代码需要补充 HMAC-SHA256 签名逻辑（默认实现未支持）
7. 点 **完成**，复制弹出的 webhook URL，形如：
   ```
   https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

### 2.2 配置到 Meego Lite

1. 打开 Meego Lite「设置 → 通知偏好」
2. 在 **飞书 Webhook** 输入框粘贴上一步复制的 URL
3. 勾选 **启用飞书通知**
4. 保存

### 2.3 验证

可以先用 curl 直接测一下 webhook 是否能工作：

```bash
curl -X POST "你的 webhook URL" \
  -H "Content-Type: application/json" \
  -d '{"msg_type":"text","content":{"text":"任务测试通知"}}'
```

- 返回 `{"code":0,"msg":"success",...}` 即成功
- 如果选了"自定义关键词"，`text` 里必须包含该关键词，否则报 `key words not found`

### 2.4 安全注意

- ⚠️ Webhook URL 等同于发消息凭证，**泄漏后任何人都能往你的群里发垃圾消息**
- 不要把 webhook 提交到 git；目前它仅存于本地 SQLite 的 `User.feishuWebhook`，风险可控
- 出问题时去群机器人设置里**重新生成 / 移除**机器人即可作废旧 webhook

## 三、企业微信通知（个人应用消息 / 群机器人）

企业微信推荐使用**自建应用消息**给个人发送通知，不依赖拉取群聊，也不需要用户在群里添加机器人。若没有配置自建应用，仍兼容原来的**群机器人 Webhook**。

### 3.1 配置企业微信自建应用（推荐）

#### 3.1.1 打开企业微信管理后台

方式一：网页登录。

1. 打开企业微信管理后台登录页：
   ```text
   https://work.weixin.qq.com/wework_admin/loginpage_wx
   ```
   也可以先打开企业微信官网：
   ```text
   https://work.weixin.qq.com/
   ```
   然后点击右上角的「企业登录 / 管理后台」。
2. 使用微信扫码登录。
3. 选择要配置的企业，进入企业微信管理后台。

方式二：从企业微信电脑客户端进入。

1. 打开企业微信电脑端。
2. 在左下角头像 / 设置入口中找到「管理企业」「企业管理后台」或类似入口。
3. 点击后会跳转到浏览器中的企业微信管理后台。

> 只有企业创建人、超级管理员或具备应用管理权限的分级管理员才能创建自建应用。普通成员通常无法进入后台，或进入后看不到「应用管理 / 创建应用」。如果没有权限，请让企业微信管理员协助创建应用，或将你添加为应用管理员。

#### 3.1.2 创建自建应用并获取配置

1. 企业微信管理员进入「管理后台 → 应用管理 → 自建应用」
2. 点击「创建应用」，应用名称可填写 `Meego Lite`
3. 创建完成后进入应用详情页，记录以下信息：
   - `AgentId`：应用 ID，对应 `.env` 中的 `WECOM_AGENT_ID`
   - `Secret`：应用密钥，对应 `.env` 中的 `WECOM_APP_SECRET`
4. 在「我的企业 → 企业信息」中找到「企业 ID」，对应 `.env` 中的 `WECOM_CORP_ID`
5. 将以下信息写入 `.env`：
   ```env
   WECOM_CORP_ID=企业ID
   WECOM_AGENT_ID=应用AgentId
   WECOM_APP_SECRET=应用Secret
   ```
6. 确认应用可见范围包含需要接收通知的成员
7. 在 Meego Lite「设置 → 通知偏好」中填写自己的**企业微信用户 ID（userid）**，并勾选**启用企业微信通知**

> 企业微信用户 ID 不是手机号，也不是昵称；可在企业微信通讯录成员详情或管理员导出的成员信息中查看。

### 3.2 个人消息发送规则

`sendNotification` 会优先使用企业微信自建应用发送个人消息：

1. `.env` 已配置 `WECOM_CORP_ID` / `WECOM_AGENT_ID` / `WECOM_APP_SECRET`
2. 用户开启 `notifyWeCom`
3. 用户填写 `wecomId`

以上条件满足时，系统会调用企业微信 `message/send` 接口给该用户发送文本消息。

### 3.3 群机器人 Webhook（兼容）

如果未配置自建应用消息，也可以继续使用企业微信群机器人 Webhook。群机器人仅能向群里发送消息，不能私发。

1. 在企业微信里找到目标群聊（没有则新建）
2. 群设置 → **群机器人 → 添加** → 新建机器人
3. 填写机器人名称（如 `Meego Lite`）和可选头像
4. 创建完成后复制 webhook URL，形如：
   ```
   https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```

> 创建群机器人**不需要管理员权限**，普通群成员也可以加。

### 3.4 配置群机器人到 Meego Lite

1. 打开 Meego Lite「设置 → 通知偏好」
2. 在 **企业微信 Webhook** 输入框粘贴 URL
3. 勾选 **启用企业微信通知**
4. 保存

### 3.5 验证群机器人 Webhook

```bash
curl -X POST "你的 webhook URL" \
  -H "Content-Type: application/json" \
  -d '{"msgtype":"text","text":{"content":"任务测试通知"}}'
```

返回 `{"errcode":0,"errmsg":"ok"}` 即成功。

### 3.6 配额限制

- 单个机器人每分钟最多 20 条消息，超过会被限流
- 单条消息正文不超过 4096 字节

## 四、用户偏好与启用方式

数据库 `User` 表中与通知相关的字段（见 `prisma/schema.prisma`）：

| 字段 | 默认值 | 说明 |
|---|---|---|
| `notifyEmail` | `true` | 是否启用邮件通知 |
| `notifyFeishu` | `false` | 是否启用飞书通知 |
| `notifyWeCom` | `false` | 是否启用企业微信通知 |
| `feishuWebhook` | `null` | 飞书自定义机器人 webhook URL |
| `wecomWebhook` | `null` | 企业微信群机器人 webhook URL |
| `wecomId` | `null` | 企业微信用户 ID，用于自建应用个人消息 |
| `leadTimeMinutes` | `60` | 截止时间提前多少分钟开始提醒 |

`sendNotification`（`src/lib/notification.ts`）会按以下规则决定向哪些渠道发送：

1. 站内通知（`in_app`）始终发送
2. 若 `notifyEmail=true` → 发送邮件（依赖 `.env` 中的 SMTP 配置）
3. 若 `notifyFeishu=true` 且填写了 `feishuWebhook` → 发送飞书卡片消息
4. 若 `notifyWeCom=true` 且填写了 `wecomId`，并且 `.env` 已配置企业微信自建应用 → 发送企业微信个人应用消息
5. 若未满足个人应用消息条件，但 `notifyWeCom=true` 且填写了 `wecomWebhook` → 发送企业微信群机器人文本消息

每条发送结果都会写入 `Notification` 表，`status` 字段为 `sent` / `failed` / `pending`，失败原因记录在 `error` 字段，可以在通知中心或数据库里查询。

## 五、常见问题

### Q1: SMTP 报错 `mail from address must be same as authorization user`
`SMTP_FROM` 中的邮箱地址不等于 `SMTP_USER`，QQ 邮箱要求两者必须一致。请把发件地址改回登录邮箱。

### Q2: SMTP 报错 `Invalid login: 535 Login Fail`
`SMTP_PASS` 填错了。请确认填的是 **16 位授权码**而非 QQ 登录密码；授权码请去掉中间的空格。

### Q3: SMTP 连接超时
- 465 端口被防火墙/公司网络封禁 → 改用 587（同时把 `SMTP_SECURE=false`）
- 仍然不行说明出口被完全限制，考虑改用其它邮件服务（如 Resend）

### Q4: 飞书 webhook 报 `key words not found`
你在创建机器人时勾选了"自定义关键词"，但发出的消息内容不包含任一关键词。要么调整代码让通知文案包含关键词，要么去机器人设置里改/取消关键词。

### Q5: 飞书 webhook 报 `sign match fail`
你勾选了"签名校验"模式，但当前代码 `sendFeishuCard` 没有计算签名。两种方案：
- 简单：去掉签名校验，改成关键词模式
- 复杂：自行在 `src/lib/notification.ts` 的 `sendFeishuCard` 中按 [飞书签名规则](https://open.feishu.cn/document/ukTMukTMukTM/ucTM5YjL3ETO24yNxkjN#348211ba) 加上 `timestamp` 与 `sign`

### Q6: 企业微信 webhook 报 `45009 reach max api daily quota limit`
单机器人每分钟 20 条的限流被触发。降低发送频率或拆分到多个机器人。

### Q7: 飞书机器人能私聊吗？
**自定义群机器人不行**，只能发群里。要给个人私发，需要走"飞书自建应用 + IM API"，需要管理员创建应用、申请权限并由用户安装。变通办法：拉一个**只有自己的群**，效果等同于私聊。

### Q8: 重启后配置没生效
- `.env` 修改后**必须重启 dev server**（`bun run dev`）才会重新加载
- 桌面应用打包产物里 `.env` 不会被自动包含，生产环境需通过 Electron 主进程注入或改成"用户在设置页填 SMTP 配置"
