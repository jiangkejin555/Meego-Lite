# 应用图标配置指南

> 本文档说明如何为 Meego Lite 桌面 App 配置自定义图标，覆盖 macOS（`.icns`）和 Windows（`.ico`）。

---

## 一、原理简介

不同平台对图标格式的要求不同：

| 平台 | 格式 | 说明 |
|------|------|------|
| macOS | `.icns` | 苹果专用图标包，内含多种尺寸（16~1024px） |
| Windows | `.ico` | 微软专用图标包，内含多种尺寸（16~256px） |
| Linux | `.png` | 多尺寸 PNG 文件 |
| 开发模式窗口/Dock | `.png` | Electron `BrowserWindow.icon` 选项 |

我们的策略是：**用一张 1024×1024 的 PNG 作为源图，自动生成上述所有格式**。

---

## 二、目录结构

打包配置约定图标放在项目根目录的 `build/` 文件夹下：

```
meego-lite/
├── build/
│   ├── icon.png      ← 1024×1024 源图
│   ├── icon.icns     ← macOS 用
│   ├── icon.ico      ← Windows 用
│   └── icons/        ← 多尺寸 PNG（Linux + 开发窗口用）
└── public/
    └── logo.svg      ← 你的原始 logo（矢量图）
```

---

## 三、生成图标的完整命令

### 步骤 1：进入项目目录

```bash
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite
```

### 步骤 2：创建 build 目录

```bash
mkdir -p build
```

### 步骤 3：把 SVG 转成 1024×1024 PNG

利用项目里已经装了的 `sharp` 库（图片处理）：

```bash
bun -e "const sharp = require('sharp'); sharp('public/logo.svg').resize(1024, 1024).png().toFile('build/icon.png').then(()=>console.log('icon.png 生成完成'))"
```

> 💡 **如果 SVG 透明区域转出来变黑**，加个白底：
>
> ```bash
> bun -e "const sharp = require('sharp'); sharp('public/logo.svg').resize(1024, 1024).flatten({background:'#ffffff'}).png().toFile('build/icon.png')"
> ```

### 步骤 4：用 electron-icon-builder 自动生成所有格式

```bash
bunx electron-icon-builder --input=build/icon.png --output=build --flatten
```

> ⚠️ **重要：electron-icon-builder 实际会把 `.icns` 和 `.ico` 生成在 `build/icons/` 子目录**，而 `package.json` 里配置的是 `build/icon.icns`，路径不一致会导致打包报错：
>
> ```
> ⨯ cannot find specified resource "build/icon.icns"
> ```
>
> **必须再补一步**，把它们移到 `build/` 根目录：
>
> ```bash
> mv build/icons/icon.icns build/icon.icns
> mv build/icons/icon.ico  build/icon.ico
> # 顺便清理工具在项目根目录生成的冗余 icons/
> rm -rf icons
> ```

执行完成后 `build/` 目录应该包含：

```
build/
├── icon.png       ← 源图（1024×1024）
├── icon.icns      ← macOS 用 ✅
├── icon.ico       ← Windows 用 ✅
└── icons/         ← 各尺寸 png（main.js 开发模式窗口用）
```

## 四、配置文件说明

### 1. `package.json` 中的 build 配置

```json
{
  "build": {
    "mac": {
      "target": "dmg",
      "category": "public.app-category.productivity",
      "identity": null,
      "icon": "build/icon.icns"
    },
    "win": {
      "target": "nsis",
      "icon": "build/icon.ico"
    },
    "dmg": {
      "icon": "build/icon.icns"
    }
  }
}
```

| 字段 | 作用 |
|------|------|
| `mac.icon` | macOS 应用 Dock / Finder / 启动台里显示的图标 |
| `win.icon` | Windows 桌面 / 任务栏 / 开始菜单里显示的图标 |
| `dmg.icon` | 双击 `.dmg` 弹出的安装窗口顶部图标 |

### 2. `electron/main.js` 窗口图标

```js
mainWindow = new BrowserWindow({
  width: 1280,
  height: 800,
  title: 'Meego Lite',
  icon: path.join(__dirname, '..', 'build', process.platform === 'win32' ? 'icon.ico' : 'icon.png'),
  webPreferences: {
    contextIsolation: true,
    nodeIntegration: false,
  },
});
```

> 这个 `icon` 主要影响 **开发模式（`bun run electron:dev`）下窗口左上角和 Dock** 的图标。打包后的应用图标由上面 `package.json` 里的 `mac.icon` / `win.icon` 决定。

## 五、重新打包验证

```bash
# 设置镜像，避免 GitHub 超时
export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/

# Mac 打包
bun run electron:build:mac

# Windows 打包（需在 Windows 电脑上执行）
bun run electron:build:win
```

打开新生成的 `dist-electron/Meego Lite-0.2.0-arm64.dmg`，安装后看 Launchpad / Dock / Finder 里的图标，应该已经是你的 logo 了。

## 六、常见坑

### 坑 1：装了新版还显示老图标（图标缓存）

macOS 有强烈的图标缓存。**装新版前先把老版本拖到废纸篓清空**。如果还不行，强制刷新缓存：

```bash
sudo rm -rfv /Library/Caches/com.apple.iconservices.store
killall Dock Finder
```

Windows 类似：删掉桌面上的快捷方式重新创建，或重启 explorer.exe。

### 坑 2：图标尺寸不够、糊

`electron-icon-builder` 要求源图至少 **1024×1024**，否则 macOS 大图模式下会糊。我们上面 `resize(1024, 1024)` 就是规避这一点。

如果你直接用现成的 PNG 而没转尺寸，可以这样校验：

```bash
file build/icon.png
# 应该输出：PNG image data, 1024 x 1024, ...
```

### 坑 3：SVG 透明背景转 PNG 变黑

有些 SVG 没有定义背景色，转 PNG 后透明区域会被填成黑色。解决：用上面"步骤 3"里的 `.flatten({background:'#ffffff'})` 加白底。

### 坑 4：想要 macOS 风格的圆角图标

macOS 系统应用图标都是带圆角的"超椭圆"形状（squircle）。如果你的 logo 是正方形：

- 简单方案：直接放正方形也能用，只是看起来跟系统应用风格不一致
- 进阶方案：用在线工具（如 https://liiir-1996.github.io/icon-pwa-tool/）生成 macOS 风格图标，下载后替换 `build/icon.icns`

### 坑 5：打包时找不到图标文件

报错：`default Electron icon is used  reason=application icon is not set` 或 `Cannot find icon file`

排查：

```bash
ls -lh build/
# 确认 icon.icns 和 icon.ico 都存在

# 确认 package.json 里的路径是相对项目根目录的，不是绝对路径
grep -A2 '"icon"' package.json
```

## 七、一键脚本（推荐）

为了避免每次手动跑这一串命令，可以把图标生成加到 `package.json` 的 scripts 里：

```json
{
  "scripts": {
    "icon:gen": "bun -e \"const sharp = require('sharp'); sharp('public/logo.svg').resize(1024, 1024).png().toFile('build/icon.png')\" && bunx electron-icon-builder --input=build/icon.png --output=build --flatten && mv build/icons/icon.icns build/icon.icns && mv build/icons/icon.ico build/icon.ico && rm -rf icons"
  }
}
```

以后只要：

```bash
bun run icon:gen
```

就能一键重新生成所有图标。如果只是更新了 `public/logo.svg`，跑这一行刷新图标后再打包即可。

---

## 八、检查清单

每次更换图标后，按下面顺序检查：

- [ ] `public/logo.svg` 已更新为最新 logo
- [ ] 跑过 `bun run icon:gen`（或手动执行步骤 3+4）
- [ ] `build/icon.icns`、`build/icon.ico`、`build/icon.png` 都存在
- [ ] `package.json` 里 `mac.icon`、`win.icon`、`dmg.icon` 路径正确
- [ ] 删掉老版应用 + 清缓存（坑 1）
- [ ] 重新打包：`bun run electron:build:mac`
- [ ] 安装新版，确认 Launchpad / Dock / Finder 图标都对了

## 总结：最短工作流

```bash
# 1. 准备好 public/logo.svg

# 2. 生成图标
mkdir -p build
bun -e "const sharp = require('sharp'); sharp('public/logo.svg').resize(1024, 1024).png().toFile('build/icon.png')"
bunx electron-icon-builder --input=build/icon.png --output=build --flatten
# ⚠️ 必须把 icns/ico 从 build/icons/ 移到 build/ 根目录
mv build/icons/icon.icns build/icon.icns
mv build/icons/icon.ico  build/icon.ico
rm -rf icons

# 3. 打包（确保 package.json 里已配置 icon 路径）
export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
bun run electron:build:mac

# 4. 装新版前清缓存（可选，看到没更新就跑）
sudo rm -rfv /Library/Caches/com.apple.iconservices.store
killall Dock Finder
```

🎉 完成！应用图标已经替换成你的 logo。
