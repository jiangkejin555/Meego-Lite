## 二、服务器初始化

### 2.1 重置密码 + SSH 登录

阿里云控制台 → 轻量应用服务器 → 重置密码 → 设置 root 密码。

```bash
# 本地终端
ssh root@<你的服务器公网IP>
```

### 2.2 安全加固

```bash
# 创建普通用户（不建议长期用 root）
adduser deploy
usermod -aG sudo deploy

# 更新系统
apt update && apt upgrade -y

# 装基础工具
apt install -y git curl vim ufw build-essential

# 防火墙
ufw allow 22
ufw allow 80
ufw allow 443
ufw enable
```

### 2.3 控制台开放端口

阿里云轻量服务器有**两层防火墙**，控制台也要开：

控制台 → 实例 → 安全 → 防火墙 → 添加规则：

- TCP 80（HTTP）
- TCP 443（HTTPS）
- TCP 22（SSH，默认已开）

### 2.4 安装运行环境

```bash
# 装 Bun（项目使用的运行时）
curl -fsSL https://bun.sh/install | bash
source ~/.bashrc

# 装 Node.js 20（兼容 Next.js 16）
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 装 Nginx
apt install -y nginx
systemctl enable nginx
systemctl start nginx

# 装 Certbot（HTTPS 证书）
apt install -y certbot python3-certbot-nginx

# 装 PM2（进程守护）
npm install -g pm2
```

### 2.5 配置 Git（拉取私有仓库）

```bash
# 生成 SSH key
ssh-keygen -t ed25519 -C "server@meego-lite"
cat ~/.ssh/id_ed25519.pub
# 复制输出，粘贴到 GitHub → Settings → SSH Keys
```

## 四、备案前临时部署（IP:8080 方案）

> ICP 备案需要 7-20 天，但在等待期间完全可以用「**公网 IP + 非 80 端口**」先把项目跑起来，做端到端验证、内部联调和 bug 修复。备案通过后再无缝切换到域名 + HTTPS。

### 4.1 为什么用 8080 端口

阿里云在国内对 **未备案的 80/443 端口**会进行拦截，但 **非 80 端口（如 8080）不受影响**：

| 端口 | 备案前 | 备案后 |
|------|--------|--------|
| 80 | ❌ 国内被封 | ✅ 可用 |
| 443 | ❌ 国内被封 | ✅ 可用 |
| **8080 / 3000 / 任意非 80** | ✅ **完全可用** | ✅ 可用 |
| 22（SSH） | ✅ 可用 | ✅ 可用 |

临时访问方式：`http://公网IP:8080`

### 4.2 修改 `.env`（用 IP 形式）

```bash
DATABASE_URL="file:/opt/Meego-Lite/prisma/prod.db"
NEXTAUTH_SECRET="$(openssl rand -base64 32)"
NEXTAUTH_URL="http://公网IP:8080"   # ⚠️ 关键：用 IP+端口，不是域名
```

### 4.3 配置 Nginx 监听 8080

新建临时配置文件：

```bash
vim /etc/nginx/sites-available/meego-lite-temp
```

写入：

```nginx
server {
    listen 8080;
    server_name _;   # 匹配所有 host（包括 IP 直连）

    gzip on;
    gzip_types text/plain text/css application/json application/javascript;

    location /_next/static/ {
        proxy_pass http://127.0.0.1:3000;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/meego-lite-temp /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx
```

### 4.4 开放 8080 端口（两层防火墙都要开）

**服务器内部防火墙**：

```bash
ufw allow 8080
```

**阿里云控制台防火墙**（必须）：

控制台 → 实例 → 安全 → 防火墙 → 添加规则：

- 协议：TCP
- 端口：`8080`
- 源：`0.0.0.0/0`

### 4.5 验证访问

浏览器打开：`http://你的公网IP:8080`

应该能看到 Meego Lite 登录页 ✅

### 4.6 注意事项与限制

| 问题 | 说明 | 影响 |
|------|------|------|
| **不能用 HTTPS** | Let's Encrypt 不给纯 IP 签证书 | 浏览器显示「不安全」，但功能正常 |
| **Cookie 安全限制** | 部分浏览器对 IP + HTTP 的 Cookie 有限制 | NextAuth 登录可能要勾「记住我」 |
| **微信/钉钉内打开会被拦** | 国内 App 不允许直连未备案 IP | 不能在企微/飞书/微信内打开测试 |
| **流量计费** | 走公网流量包 | 200GB/月，自测足够 |

### ✅ 适合的场景

- 自己 + 团队内部测试（直接用浏览器访问）
- 验证部署流程是否完整
- 调试 Prisma / NextAuth / SMTP 等环境问题
- 做完整的端到端 demo

### ❌ 不适合的场景

- 给客户/外部用户演示（HTTP 不安全提示劝退）
- 微信/飞书内分享链接（会被拦）
- 长期生产使用

### 4.7 备案通过后的迁移（5 分钟搞定）

**Step 1：改 `.env`**

```bash
NEXTAUTH_URL="https://你的域名.com"
```

**Step 2：切换 Nginx 配置**

```bash
# 禁用临时配置
rm /etc/nginx/sites-enabled/meego-lite-temp

# 启用正式配置（监听 80，下一章会创建）
ln -s /etc/nginx/sites-available/meego-lite /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

**Step 3：配置 HTTPS**

```bash
certbot --nginx -d 你的域名.com -d www.你的域名.com
```

**Step 4：重启应用**

```bash
pm2 restart meego-lite
```

**Step 5：关闭临时端口（可选，更安全）**

```bash
ufw delete allow 8080
# 阿里云控制台同步删掉 8080 规则
```

## 五、域名解析 + Nginx + HTTPS（备案通过后）

### 5.1 域名解析

阿里云控制台 → 域名 → 解析设置 → 添加记录：

| 记录类型 | 主机记录 | 记录值 | TTL |
|---------|---------|--------|-----|
| A | `@` | 服务器公网 IP | 600 |
| A | `www` | 服务器公网 IP | 600 |

### 5.2 配置 Nginx 反向代理

```bash
vim /etc/nginx/sites-available/meego-lite
```

写入：

```nginx
server {
    listen 80;
    server_name 你的域名.com www.你的域名.com;

    # Gzip 压缩，把 3M 带宽榨干
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1024;

    # 静态资源长期缓存
    location /_next/static/ {
        proxy_pass http://127.0.0.1:3000;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    # 上传/请求体大小（如以后有文件上传）
    client_max_body_size 20M;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/meego-lite /etc/nginx/sites-enabled/
rm /etc/nginx/sites-enabled/default
nginx -t
systemctl reload nginx
```

### 5.3 配置 HTTPS（备案通过后再做）

```bash
certbot --nginx -d 你的域名.com -d www.你的域名.com
# 按提示填邮箱、同意协议、选择强制 HTTPS

# 测试自动续期
certbot renew --dry-run
```

## 七、性能优化与容量规划（多用户场景）

### 7.1 容量评估：2核2G·3M 能扛多少用户

> 决定"卡不卡"的核心是**同时在线人数**和**QPS**，不是注册总数。

**承载能力一览**：

| 同时在线 | 体验 | 服务器状态 |
|---------|------|----------|
| 1-10 人 | 秒开 | CPU < 10%，无感 |
| 10-30 人 | 流畅 | CPU 20-40%，正常 |
| 30-50 人 | 偶尔慢 | 带宽吃紧，首屏 3-5 秒 |
| 50-80 人 | 明显卡顿 | 内存接近上限 |
| 80+ 人 | 频繁超时 | 可能 502/OOM |

**典型场景适配**：

| 团队规模 | 注册用户 | DAU | 同时在线峰值 | 是否够用 |
|---------|---------|------|-------------|---------|
| 5 人小组 | 5 | 5 | 5 | ✅ 完美 |
| 1 个部门 | 30 | 20 | 10 | ✅ 流畅 |
| 1 个公司（小） | 100 | 50 | 20 | ✅ 够用 |
| 1 个公司（中） | 300 | 150 | 40 | ⚠️ 需要优化 |
| 多公司租户 | 1000+ | 500+ | 100+ | ❌ 升级硬件 |

**主要瓶颈排序**：

1. **带宽 3M（最先满）**：首屏 400-700KB，每秒最多支撑 1-2 人首次加载
2. **内存 2GB**：Next.js + Bun 占用约 600MB，余量支撑 ~120 个并发请求
3. **CPU 2 核**：理论 130 QPS，通常不是瓶颈
4. **SQLite 写锁**：写 QPS 上限 50-200，读不限

### 7.2 优化策略一：Nginx Brotli 压缩（提升约 20%）

Brotli 压缩比 Gzip 高 15-25%，特别适合带宽受限场景。

**安装 Brotli 模块**：

```bash
sudo apt install -y nginx-module-brotli
```

**修改 `/etc/nginx/nginx.conf`，在 `http {` 块外加载模块**：

```nginx
load_module modules/ngx_http_brotli_filter_module.so;
load_module modules/ngx_http_brotli_static_module.so;
```

**在 `http {}` 块内开启 Brotli**：

```nginx
brotli on;
brotli_comp_level 6;
brotli_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript image/svg+xml;
brotli_min_length 1024;
```

**重载配置**：

```bash
nginx -t && systemctl reload nginx
```

**验证生效**：

```bash
curl -H "Accept-Encoding: br" -I https://你的域名.com
# 响应头应包含：Content-Encoding: br
```

### 7.3 优化策略二：Cloudflare 免费 CDN（提升 3-5 倍）⭐

**这是最重要的优化**，等于把 3M 带宽放大到 30M+ 体感。

**前置条件**：域名已备案（Cloudflare 套国内服务器需要域名备案）。

**操作流程**：

1. 注册 Cloudflare 账号：<https://www.cloudflare.com>
2. Add Site → 输入你的域名 → 选 Free 套餐
3. 把阿里云域名的 NS（域名服务器）改成 Cloudflare 给的两个：
   - 阿里云 → 域名管理 → DNS 修改 → 改为 Cloudflare 的 NS
4. 等待 DNS 生效（10 分钟到 24 小时）
5. 在 Cloudflare 控制台添加 DNS 记录：
   - A `@` → 服务器 IP（**橙色云朵开启代理**）
   - A `www` → 服务器 IP（橙色云朵）
6. SSL/TLS → 选 **Full**（不是 Full Strict）
7. Speed → Optimization → 全部开启（Auto Minify、Brotli、Rocket Loader 可选）
8. Caching → Configuration → Browser Cache TTL 设为 1 年

**注意**：

- 国内访问 Cloudflare 速度一般，但**静态资源命中缓存后会从你服务器卸载**，主站速度反而变快
- 服务器 Nginx 配置里需要拿真实 IP：

```nginx
# 加在 server {} 块内
real_ip_header CF-Connecting-IP;
set_real_ip_from 0.0.0.0/0;
```

### 7.4 优化策略三：SQLite WAL 模式（写性能 +5 倍）

SQLite 默认是回滚日志模式，WAL（Write-Ahead Logging）能大幅提升并发写入性能。

**修改方式**（任选其一）：

**方式 A：在 Prisma 启动时执行（推荐）**

在项目根目录创建 `prisma/init.sql`：

```sql
PRAGMA journal_mode=WAL;
PRAGMA synchronous=NORMAL;
PRAGMA cache_size=10000;
PRAGMA temp_store=MEMORY;
```

每次部署后执行一次：

```bash
sqlite3 /opt/Meego-Lite/prisma/prod.db < /opt/Meego-Lite/prisma/init.sql
```

**方式 B：直接命令行设置**

```bash
sqlite3 /opt/Meego-Lite/prisma/prod.db "PRAGMA journal_mode=WAL;"
sqlite3 /opt/Meego-Lite/prisma/prod.db "PRAGMA synchronous=NORMAL;"
```

**验证生效**：

```bash
sqlite3 /opt/Meego-Lite/prisma/prod.db "PRAGMA journal_mode;"
# 应该返回 wal
```

> ⚠️ 启用 WAL 后，数据库目录会多出 `prod.db-wal` 和 `prod.db-shm` 两个文件，备份时三个文件都要备份。

### 7.5 优化策略四：Next.js 静态资源缓存

Nginx 配置中已加 `_next/static/` 长期缓存，再补一条图片缓存：

```nginx
# 在 server {} 内追加
location ~* \.(jpg|jpeg|png|gif|ico|webp|svg|woff|woff2)$ {
    proxy_pass http://127.0.0.1:3000;
    expires 30d;
    add_header Cache-Control "public, no-transform";
}
```

### 7.6 监控：判断什么时候该升级

定期跑下面命令观察资源水位：

```bash
# CPU 和内存
pm2 monit

# 整体内存
free -h

# 磁盘 IO
iostat -x 2 5

# 实时连接数
ss -s

# Nginx 当前连接
ss -tn | grep :80 | wc -l
```

**触发升级的红线**（任意 2 条命中就该升级）：

- [ ] CPU 持续 > 70%
- [ ] 内存使用 > 1.5GB
- [ ] 带宽常态打满 3M（阿里云控制台流量监控）
- [ ] 首屏加载 > 5 秒
- [ ] 经常出现 502 / 504

### 7.7 升级路径

| 当前 | 下一步 | 价格 | 能扛 |
|------|-------|------|-----|
| 2核2G·3M·40G | 2核4G·5M·60G | ¥198/年 | 100 同时在线 |
| 2核4G·5M·60G | 4核8G·8M·100G | ¥600/年 | 300 同时在线 |
| 单机 SQLite | + 阿里云 RDS PostgreSQL | ¥1500+/年 | 解除写锁瓶颈 |
| 单机方案 | 多机 + RDS + SLB | ¥3000+/年 | 1000+ 同时在线 |

**SQLite → PostgreSQL 迁移触发条件**：

- 同时在线 > 100，写操作频繁
- 数据量 > 10GB
- 需要多机部署做高可用

迁移时只需修改 `prisma/schema.prisma` 的 `provider` 和 `DATABASE_URL`，schema 几乎不用改。

## 八、常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 域名打不开 | 备案没过 / 解析未生效 | 等备案通过；`nslookup 域名` 查解析 |
| 502 Bad Gateway | Node 进程挂了 | `pm2 logs` 看错误，`pm2 restart` |
| HTTPS 证书报错 | 80 端口没开 | 检查 ufw + 阿里云控制台防火墙 |
| 登录态丢失 | `NEXTAUTH_URL` 不对 | 改成 `https://域名.com` 后重启 |
| Prisma 报错 | 二进制缺失 | 删 `node_modules` 重装 |
| 网页打开很慢 | 带宽吃紧 | Nginx 开 Gzip + 静态资源走 CDN |
