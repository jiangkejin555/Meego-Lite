## 二、首次部署（一条命令搞定）

### 2.1 准备工作

#### Step 1：阿里云控制台开放端口

控制台 → 实例 → 安全 → 防火墙 → 添加规则：

- TCP 22（SSH，默认已开）
- TCP 80（HTTP，备案后用）
- TCP 443（HTTPS，备案后用）
- **TCP 8080**（临时访问，备案前必须开）

#### Step 2：SSH 登录服务器

```bash
ssh admin@<你的公网IP>
```

#### Step 3：克隆项目

```bash
mkdir -p ~/kejin && cd ~/kejin
git clone git@github.com:jiangkejin555/Meego-Lite.git
cd Meego-Lite
```

> 💡 私有仓库需要先在服务器上配 SSH key：
> ```bash
> ssh-keygen -t ed25519 -C "server@meego-lite"
> cat ~/.ssh/id_ed25519.pub
> # 把输出加到 GitHub → Settings → SSH Keys
> ```

#### Step 4：配置环境变量

```bash
cp .env.example .env
vim .env
```

**最关键的几个变量**（必填）：

```bash
# 数据库（绝对路径更稳）
DATABASE_URL="file:/home/admin/kejin/Meego-Lite/prisma/prod.db"

# NextAuth + JWT 密钥（用 openssl rand -base64 32 生成）
NEXTAUTH_SECRET="..."
NEXTAUTH_URL="http://公网IP:8080"   # 备案后改成 https://域名.com
AUTH_SECRET="..."                    # 多用户登录必填，再生成一次

# SMTP 邮箱（验证码登录必填）
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=your_account@qq.com
SMTP_PASS=16位QQ邮箱授权码
SMTP_FROM="Meego Lite <your_account@qq.com>"
```

> 💡 生成密钥：`openssl rand -base64 32`
>
> 💡 QQ 邮箱授权码获取：QQ 邮箱 → 设置 → 账户 → 开启 IMAP/SMTP 服务

### 2.2 一键部署 ⭐

```bash
make first-deploy
```

这一条命令自动完成：

```
1. system-deps      装系统依赖（unzip / nginx / certbot / sqlite3）
                    + 装 Node.js 20、Bun、PM2
                    + 关闭 needrestart 弹窗
2. swap             创建 2GB Swap（防 2G 内存 OOM）
3. registry-config  配置 Bun/npm 国内镜像（npmmirror）
4. install          按 bun.lock 严格装项目依赖（生产模式，跳过 electron）
5. build            生成 Prisma Client + 构建 Next.js
6. db-init          初始化 prod.db + 启用 WAL 模式
7. nginx-temp       配置临时 Nginx（监听 8080）
8. start            PM2 启动应用 + 开机自启
```

### 2.3 验证部署

部署完成后，终端会输出访问地址：

```
✓ 首次部署完成！
  访问地址：http://<公网IP>:8080
  查看日志：make logs
  备案通过后开 HTTPS：make ssl DOMAIN=你的域名.com
```

浏览器访问 `http://<公网IP>:8080`，看到 Meego Lite 登录页 ✅

### 2.4 配置 PM2 开机自启（一次性）

`make first-deploy` 已自动跑了 `pm2 save`，但还需要一次性配置 systemd 开机自启：

```bash
pm2 startup
# 复制输出的命令并执行（一般是一行 sudo env PATH=...）
```

之后服务器重启会**自动**恢复 Meego Lite 服务。

## 三、备案前临时访问（IP:8080）

### 3.1 为什么用 8080？

阿里云对未备案的 80/443 端口会拦截，但 8080 不受影响。

| 端口 | 备案前 | 备案后 |
|------|--------|--------|
| 80 / 443 | ❌ 国内被封 | ✅ 可用 |
| **8080** | ✅ **完全可用** | ✅ 可用 |

### 3.2 已自动配置

`make first-deploy` 内部已调用 `make nginx-temp`，自动：

- 创建 `/etc/nginx/sites-available/meego-lite-temp`
- 监听 8080 → 反代到 127.0.0.1:3000
- 开放 8080 防火墙

### 3.3 限制说明

| 问题 | 说明 |
|------|------|
| 不能用 HTTPS | Let's Encrypt 不给 IP 签证书 |
| 微信/钉钉打开会被拦 | 国内 App 不允许直连未备案 IP |
| 浏览器显示"不安全" | HTTP 明文，但功能正常 |

**适合场景**：内部测试、开发联调、邀请熟人体验。

## 四、备案通过后启用 HTTPS

### 4.1 域名解析

阿里云控制台 → 域名 → 解析设置：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | `@` | 公网 IP |
| A | `www` | 公网 IP |

### 4.2 一键 HTTPS ⭐

```bash
make ssl DOMAIN=你的域名.com
```

自动完成：

```
1. nginx-prod       配置正式 Nginx（监听 80）
                    + Gzip 压缩
                    + 静态资源缓存（_next/static 缓存 1 年，图片 30 天）
                    + 删除临时 8080 配置
2. certbot          DNS 验证 → 申请 Let's Encrypt 证书
                    + 自动改 Nginx 加上 443 监听
                    + 强制 HTTP → HTTPS 跳转
3. renew --dry-run  测试自动续期（每 90 天自动续）
```

### 4.3 更新环境变量

```bash
vim .env
# 改: NEXTAUTH_URL="https://你的域名.com"

make restart
```

### 4.4 关闭 8080（可选，更安全）

```bash
sudo ufw delete allow 8080
# 阿里云控制台同步删掉 8080 规则
```

## 五、日常运维与发版

### 5.1 一键发版 ⭐

**本地 Mac 上**：

```bash
make remote-deploy
```

**或者 SSH 到服务器**：

```bash
make update
```

`make update` 自动完成：

```
1. db-backup    备份当前数据库到 /opt/backup/prod-时间戳.db
2. git pull     拉取最新代码
3. db-push      同步 schema.prisma 到数据库（保留已有数据）
                内部: prisma generate + prisma db push
4. build        装依赖 + 重新构建
5. restart      pm2 restart meego-lite
```

### 5.2 不同场景的命令选择

| 场景 | 命令 | 说明 |
|------|------|------|
| 完整发版 | `make update` ⭐ | 备份 → pull → db-push → build → restart |
| 简版部署（不拉代码、不备份） | `make deploy` | db-push + build + restart |
| 只更新数据库表结构 | `make db-push` | 改了 `schema.prisma` 时用 |
| 只重启服务 | `make restart` | 配置改了不需要构建时用 |
| 只重新构建 | `make build && make restart` | 代码改了但 db 没变 |
| 手动备份数据库 | `make db-backup` | 重大变更前手动备份 |

### 5.3 监控与日志

```bash
make status      # PM2 进程状态
make logs        # 实时日志（按 Ctrl+C 退出）
make monit       # CPU / 内存实时监控
```

### 5.4 应用控制

```bash
make start       # 启动
make stop        # 停止
make restart     # 重启
```

### 5.5 Schema 改动的开发约定

每次修改 `prisma/schema.prisma`（加字段/加表），**必须同时更新** `src/lib/db-migrations.ts`：

```typescript
// 1. schema.prisma 加字段
model User {
  newField String?
}

// 2. db-migrations.ts 加对应迁移函数（Electron 自动迁移用）
export async function ensureUserNewFieldColumn(client: RawSqlDb) {
  // 检查列存在，不存在则 ALTER TABLE ADD COLUMN
}

// 3. 在 ensureDatabaseSchema() 里调用
await ensureUserNewFieldColumn(db);
```

服务器走标准 `prisma db push`，Electron 走运行时自动迁移，**两边都覆盖**。

## 七、运行架构与端口

### 7.1 整体架构

```
┌──────────────────────────────────────────────┐
│  用户浏览器                                    │
│  http://公网IP:8080  或  https://你的域名.com  │
└────────────────┬─────────────────────────────┘
                 │ HTTPS / HTTP
                 ↓
┌──────────────────────────────────────────────┐
│  阿里云轻量服务器                             │
│  ┌────────────────────────────────────────┐  │
│  │ Nginx (端口 80/8080/443)                │  │
│  │  ├─ 反向代理                            │  │
│  │  ├─ Gzip 压缩                          │  │
│  │  └─ 静态资源缓存                        │  │
│  └────────────┬───────────────────────────┘  │
│               │ proxy_pass 127.0.0.1:3000    │
│               ↓                                │
│  ┌────────────────────────────────────────┐  │
│  │ PM2 (进程守护) + 开机自启                │  │
│  │  └─ Bun 进程 (端口 3000)                │  │
│  │     └─ Next.js Standalone Server       │  │
│  │        └─ Prisma Client                 │  │
│  │           └─ SQLite (prod.db) ← 数据库   │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

### 7.2 各组件职责

| 组件 | 职责 |
|------|------|
| **Nginx** | 反向代理 / HTTPS / Gzip / 静态缓存 |
| **PM2** | 进程守护 / 崩溃重启 / 开机自启 / 日志管理 |
| **Bun** | JavaScript 运行时（替代 Node.js）|
| **Next.js Standalone** | Web 应用，最小化运行时 |
| **Prisma** | ORM，类型安全的数据库访问 |
| **SQLite** | 单文件数据库（`prod.db`）|

### 7.3 关键端口

| 端口 | 监听者 | 用途 | 是否对外 |
|------|-------|------|---------|
| 22 | sshd | SSH 远程登录 | ✅ 公网 |
| 80 | Nginx | HTTP | ✅ 公网（备案后）|
| 443 | Nginx | HTTPS | ✅ 公网（备案后）|
| **8080** | Nginx | 临时 HTTP | ✅ 公网（备案前）|
| **3000** | Bun/Next.js | 应用本身 | ❌ 仅 127.0.0.1 |

> ⚠️ **3000 端口只允许本机访问**，外部请求必须经 Nginx 转发，确保安全。

## 八、性能优化（多用户场景）

### 8.1 容量评估：2核2G·3M 能扛多少用户

| 同时在线 | 体验 | 服务器状态 |
|---------|------|----------|
| 1-10 人 | 秒开 | CPU < 10% |
| 10-30 人 | 流畅 | CPU 20-40% |
| 30-50 人 | 偶尔慢 | 带宽吃紧 |
| 50-80 人 | 明显卡顿 | 内存接近上限 |
| 80+ 人 | 频繁超时 | 需升级 |

### 8.2 优化策略一：Cloudflare 免费 CDN（最有效）⭐

**等于把 3M 带宽放大到 30M+ 体感**。

1. 注册 <https://www.cloudflare.com>
2. Add Site → 输入域名 → Free 套餐
3. 阿里云域名 NS 改为 Cloudflare 的两个
4. Cloudflare 控制台添加 DNS 记录：
   - A `@` → 公网 IP（**橙色云朵开启代理**）
   - A `www` → 公网 IP（橙色云朵）
5. SSL/TLS → 选 **Full**
6. Speed → Optimization → 全部开启

**前置条件**：域名已备案。

### 8.3 优化策略二：Brotli 压缩（提升 20%）

```bash
sudo apt install -y nginx-module-brotli
```

修改 `/etc/nginx/nginx.conf`，在 `http {}` 块外加载模块：

```nginx
load_module modules/ngx_http_brotli_filter_module.so;
load_module modules/ngx_http_brotli_static_module.so;
```

`http {}` 内开启：

```nginx
brotli on;
brotli_comp_level 6;
brotli_types text/plain text/css application/json application/javascript image/svg+xml;
```

### 8.4 优化策略三：SQLite WAL 模式（写性能 +5 倍）

`make db-init` 已自动开启 WAL，验证：

```bash
sqlite3 prisma/prod.db "PRAGMA journal_mode;"
# 应该返回 wal
```

### 8.5 监控水位

```bash
make monit       # PM2 实时监控
free -h          # 内存
df -h            # 磁盘
```

**升级红线**（任意 2 条命中就升级）：

- [ ] CPU 持续 > 70%
- [ ] 内存使用 > 1.5GB
- [ ] 带宽常态打满 3M
- [ ] 首屏加载 > 5 秒
- [ ] 经常 502 / 504

### 8.6 升级路径

| 当前 | 下一步 | 价格 | 能扛 |
|------|-------|------|-----|
| 2核2G·3M | 2核4G·5M | ¥198/年 | 100 同时在线 |

## 九、数据备份与恢复

### 9.1 自动备份（推荐）

`make update` 已自动备份数据库到 `/opt/backup/prod-时间戳.db`。

### 9.2 手动备份

```bash
make db-backup
```

### 9.3 定时备份（每天凌晨 3 点）

```bash
crontab -e
```

加一行：

```bash
0 3 * * * cp /home/admin/kejin/Meego-Lite/prisma/prod.db /opt/backup/prod-$(date +\%Y\%m\%d).db && find /opt/backup -mtime +30 -delete
```

> 保留近 30 天历史，自动清理过期备份。

### 9.4 紧急恢复

```bash
# 1. 停服务
make stop

# 2. 恢复备份
sudo cp /opt/backup/prod-XXXX.db /home/admin/kejin/Meego-Lite/prisma/prod.db
sudo chown admin:admin /home/admin/kejin/Meego-Lite/prisma/prod.db

# 3. 启动
make start
```

## 十、常见问题排查

### Q1：`make first-deploy` 弹 `Daemons using outdated libraries` 弹窗

Makefile 已自动关闭 needrestart 弹窗。如果仍出现，按 **Tab → Enter** 确认 OK 即可。

### Q2：`bun install` 报 `ConnectionRefused` / `ETIMEDOUT`

国内访问海外 npm 超时。Makefile 已自动配国内镜像 + npm 兜底，正常情况下不会出现。

如果仍报错：

```bash
make registry-config
make install
```

### Q3：`bun install` 报 `lockfile is frozen`

清理 lockfile 重新生成：

```bash
rm -f bun.lock bun.lockb
make install
```

### Q4：构建报 `Failed to fetch Geist from Google Fonts`

服务器无法访问 Google。项目已用系统字体替代 Google Fonts，确保拉取了最新代码：

```bash
git pull
make build
```

### Q5：访问不了 80 端口

未备案。临时用 `http://公网IP:8080`（Makefile 已自动配置）。

### Q6：`make ssl` 报错

确认：
- 域名已备案
- DNS 解析已生效（`nslookup 你的域名.com`）
- 80 端口在阿里云控制台已开放
- 服务器 ufw 也开放了 80：`sudo ufw allow 80`

### Q7：服务器重启后服务没自动恢复

跑过 `pm2 startup` 吗？没跑就执行：

```bash
pm2 startup
# 复制输出的命令并执行（带 sudo env PATH=... 那行）
pm2 save
```

### Q8：发版后服务报错 schema 不匹配

`make update` 已含 `db-push`，应该不会发生。手动同步：

```bash
make db-push
make restart
```

### Q9：邮箱验证码收不到

检查 `.env` 里的 SMTP：

- `SMTP_PASS` 是邮箱**授权码**，不是登录密码
- 阿里云服务器**默认封禁 25 端口**，必须用 465 + `SMTP_SECURE=true`
- QQ 邮箱：邮箱设置 → 账户 → 开启 IMAP/SMTP 服务

### Q10：想看应用真实性能

```bash
make monit       # PM2 实时监控
make logs        # 应用日志

# Nginx 日志
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

## 十一、推荐部署时间线

```text
Day 1 上午（约 30 分钟）
  ├─ 阿里云买服务器 + 买域名
  ├─ 提交 ICP 备案
  ├─ 阿里云控制台开放 22/80/443/8080 端口
  └─ SSH 登录服务器

Day 1 下午（约 30 分钟）
  ├─ git clone Meego-Lite
  ├─ 配置 .env
  ├─ make first-deploy   ← 一条命令搞定 ✅
  ├─ pm2 startup         ← 配置开机自启
  └─ 通过 http://公网IP:8080 访问 ✅

Day 2-15（与备案并行）
  └─ 持续迭代代码、修 bug
     - 本地: git push
     - 服务器: make remote-deploy 或 ssh + make update

备案通过当天（约 15 分钟）
  ├─ 阿里云域名 → 解析 A 记录到公网 IP
  ├─ make ssl DOMAIN=你的域名.com  ← 一条命令开 HTTPS ✅
  ├─ vim .env 改 NEXTAUTH_URL=https://...
  ├─ make restart
  └─ 通过 https://你的域名.com 正式上线 🎉

正式上线后
  ├─ 配 Cloudflare 免费 CDN（提升 3-5 倍速度）
  ├─ 设置 cron 每天 3 点自动备份
  └─ 持续 make remote-deploy 迭代
```

## 附录：快速命令速查

```bash
# ============= 一键命令 =============
make help                       # 所有命令列表
make first-deploy               # 首次部署
make update                     # 日常发版
make ssl DOMAIN=xxx.com         # 备案后开 HTTPS
make remote-deploy              # 本地远程部署

# ============= 常用维护 =============
make restart                    # 重启
make logs                       # 看日志
make status                     # 看状态
make monit                      # 资源监控
make db-backup                  # 备份数据库
make db-push                    # 同步表结构

# ============= 运维原始命令 =============
pm2 ls                          # PM2 进程列表
pm2 logs meego-lite             # 实时日志
sudo systemctl status nginx     # Nginx 状态
sudo nginx -t                   # 测试配置
sudo systemctl reload nginx     # 重载配置
sudo certbot renew              # 手动续 HTTPS
curl ifconfig.me                # 看公网 IP
free -h                         # 看内存
df -h                           # 看磁盘
```

## 总结

| 阶段 | 命令 | 耗时 |
|------|------|------|
| 首次部署 | `make first-deploy` | 5-10 分钟 |
| 日常发版 | `make update` 或 `make remote-deploy` | 1-3 分钟 |
| 开 HTTPS | `make ssl DOMAIN=xxx.com` | 1 分钟 |
| 紧急恢复 | `make stop && cp 备份 && make start` | 1 分钟 |

**核心理念**：所有运维操作通过 `Makefile` 一键完成，让部署像 npm 命令一样简单。
