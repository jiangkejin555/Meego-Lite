## 一、运行方式总览

| 方式 | 场景 | 数据库 | 是否需要域名 | 难度 | 推荐度 |
|------|------|-------|------------|------|-------|
| **A. 本地开发** | 个人体验、二次开发 | SQLite (dev.db) | ❌ | ⭐ | ⭐⭐⭐⭐⭐ |
| **B. 本地生产** | 个人长期使用 | SQLite (dev.db) | ❌ | ⭐ | ⭐⭐⭐⭐ |
| **C. Docker** | 自有服务器/NAS | SQLite (容器卷) | 可选 | ⭐⭐ | ⭐⭐⭐ |
| **D. 阿里云服务器** ⭐ | 团队协作、对外服务 | SQLite (prod.db) | ✅ 需备案 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **E. 桌面 App** | 给非技术用户分发 | SQLite (用户目录) | ❌ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **F. Vercel + Neon** | 海外用户、零运维 | PostgreSQL (云) | ✅ 自带 | ⭐⭐ | ⭐⭐⭐ |

### 怎么选？

- **个人电脑临时用** → 方式 A
- **个人长期当工作台用** → 方式 B
- **公司内网共享** → 方式 C 或 D
- **对外提供服务（团队/客户）** → **方式 D（阿里云）⭐**
- **给同事发独立 App 用** → 方式 E
- **需要 Serverless / 海外用户** → 方式 F

## 二、准备工作

### 1. 通用依赖

| 工具 | 版本 | 用途 |
|------|-----|------|
| Bun | ≥ 1.1 | JavaScript 运行时 |
| Git | 任意 | 拉取代码 |
| Node.js | ≥ 18（可选）| 部分场景兜底 |

### 2. 安装 Bun（必装）

**Mac/Linux**：

```bash
curl -fsSL https://bun.sh/install | bash
# 重启终端后验证
bun --version
```

**Windows**：

```powershell
powershell -c "irm bun.sh/install.ps1 | iex"
```

### 3. 拉取项目

```bash
git clone git@github.com:jiangkejin555/Meego-Lite.git
cd Meego-Lite
```

### 4. 配置环境变量

```bash
cp .env.example .env
# 用编辑器打开 .env，按需修改
```

最关键的几个变量：

```bash
DATABASE_URL="file:./prisma/dev.db"        # 本地用 SQLite
NEXTAUTH_SECRET="..."                       # 用 openssl rand -base64 32 生成
NEXTAUTH_URL="http://localhost:3000"
AUTH_SECRET="..."                           # 多用户登录必填，再生成一次
SMTP_HOST=smtp.qq.com                       # 验证码邮件用，QQ/163 都可
SMTP_USER=your_account@qq.com
SMTP_PASS=16位邮箱授权码
```

## 三、方式 A：本地开发模式（个人使用）

> 启动最快，适合个人体验或二次开发。

### 步骤

```bash
# 1. 进入项目
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite

# 2. 装依赖（首次）
bun install

# 3. 初始化数据库（首次）
bun run db:generate
bun run db:push

# 4. 启动开发服务器
bun run dev
```

启动成功后浏览器访问：

```
http://localhost:3000
```

### 日常使用

```bash
cd Meego-Lite
bun run dev
```

按 `Ctrl + C` 停止。

## 四、方式 B：本地生产模式（持续后台运行）

> 适合个人长期把它当工作台用，关掉终端不停服务。

### 步骤

```bash
# 1. 装 PM2（进程守护）
bun add -g pm2

# 2. 构建生产版本
bun run build

# 3. PM2 后台启动
pm2 start "bun .next/standalone/server.js" --name meego-lite \
  --env NODE_ENV=production \
  --env HOSTNAME=127.0.0.1 \
  --env PORT=3000

# 4. 设置开机自启
pm2 save
pm2 startup
```

### PM2 常用命令

| 命令 | 作用 |
|------|------|
| `pm2 ls` | 查看运行状态 |
| `pm2 logs meego-lite` | 看实时日志 |
| `pm2 restart meego-lite` | 重启 |
| `pm2 stop meego-lite` | 停止 |
| `pm2 monit` | 实时监控 CPU/内存 |

## 五、方式 C：Docker 运行（团队/服务器自部署）

> 环境隔离、一键启动，适合 NAS、内网服务器、有自有云的场景。

### 步骤

```bash
# 1. 装 Docker Desktop
# https://www.docker.com/products/docker-desktop/

# 2. 进入项目
cd Meego-Lite

# 3. 一键启动
docker compose up -d --build
```

访问 `http://localhost:3000` 即可使用。

### Docker 常用命令

| 命令 | 作用 |
|------|------|
| `docker compose ps` | 查看容器状态 |
| `docker compose logs -f` | 实时日志 |
| `docker compose stop` | 暂停（数据保留）|
| `docker compose start` | 启动 |
| `docker compose down` | 停止 + 清理容器（数据保留 ✅）|
| `docker compose down -v` | ⚠️ **删除数据卷，会丢数据，慎用** |

## 六、方式 D：阿里云服务器一键部署（推荐生产）⭐

> 团队协作、对外服务的最佳选择。完整方案见 [阿里云部署完整方案.md](./阿里云部署完整方案.md)。

### 6.1 准备资源

| 资源 | 推荐配置 | 价格 |
|------|---------|------|
| 阿里云轻量服务器 | 2核2G·3M·40G SSD | ¥99/年（首年）|
| 域名（.com） | 任选 | ¥55-75/年 |
| ICP 备案 | 个人/企业 | 免费，等 7-20 天 |

### 6.2 一键部署（使用 Makefile）

项目已内置 `Makefile`，从环境到上线一条命令搞定：

```bash
# 1. SSH 登录服务器
ssh admin@<你的公网IP>

# 2. 拉项目
cd /opt && git clone git@github.com:jiangkejin555/Meego-Lite.git
cd Meego-Lite

# 3. 配置环境变量
cp .env.example .env
vim .env
# 改 DATABASE_URL 为绝对路径：file:/opt/Meego-Lite/prisma/prod.db
# 填 NEXTAUTH_SECRET / AUTH_SECRET / SMTP_*

# 4. 一键部署（自动装环境 + Swap + 镜像 + 构建 + Nginx + 启动）
make first-deploy
```

完成后访问：

```
http://<公网IP>:8080
```

> ⚠️ 备案前用 8080 端口（80 端口未备案会被拦截）

### 6.3 备案通过后启用 HTTPS

```bash
make ssl DOMAIN=你的域名.com
```

自动完成：DNS 校验 → 申请证书 → 改 Nginx → 强制 HTTPS。

### 6.4 日常更新（一条命令）

```bash
# 在本地 Mac 上
make remote-deploy

# 或登录服务器执行
ssh admin@<IP> "cd /opt/Meego-Lite && make update"
```

`make update` 自动完成：备份数据库 → 拉代码 → 同步表结构 → 构建 → 重启。

### 6.5 Makefile 全部命令

```bash
make help              # 查看所有命令
make first-deploy      # 首次部署
make update            # 日常更新
make deploy            # 简版部署（不拉代码）
make remote-deploy     # 本地 → 服务器一键部署

make db-init           # 初始化数据库（首次）
make db-push           # 同步 schema 到数据库（改 schema 后）
make db-backup         # 手动备份数据库

make start / stop / restart / status / logs / monit
make ssl DOMAIN=xxx    # 备案后开 HTTPS
```

## 七、方式 E：打包成桌面 App（Mac/Win 分发）

> 适合给非技术用户/客户分发独立 App。

### 7.1 原理

```
双击 App
  ↓
Electron 启动 → 内置 Chromium + Node.js
  ↓
首次启动复制 prisma/dev.db 到用户数据目录
  ↓
fork Node 子进程跑 Next.js standalone server (127.0.0.1:3000)
  ↓
原生窗口加载 http://127.0.0.1:3000
```

**数据存储位置**（每台电脑独立一份）：

| 系统 | 数据库路径 |
|------|----------|
| macOS | `~/Library/Application Support/Meego Lite/meego-lite.db` |
| Windows | `%APPDATA%\Meego Lite\meego-lite.db` |
| Linux | `~/.config/Meego Lite/meego-lite.db` |

> ⚠️ **多人使用时数据不互通**，需要协作请用方式 D。

### 7.2 打包步骤

#### 国内网络优化（必做）

```bash
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
```

#### Mac 平台

```bash
bun run electron:build:mac
```

产物：`dist-electron/Meego Lite-x.x.x-arm64.dmg`（Apple Silicon）和 `*.dmg`（Intel）。

#### Windows 平台（需在 Windows 电脑上）

```bash
bun run electron:build:win
```

产物：`dist-electron/Meego Lite Setup x.x.x.exe`。

> 💡 跨平台限制：Mac 不能打 Windows 包，反之亦然。建议用 GitHub Actions CI 同时打两个平台。

### 7.3 用户安装

#### Mac

1. 双击 `.dmg` → 拖到"应用程序"
2. **首次打开会提示"无法验证开发者"**：
   - macOS 14-：右键 → 打开
   - macOS 15+：系统设置 → 隐私与安全性 → 仍要打开
3. 提示"App 已损坏"时，用户在终端执行：
   ```bash
   xattr -dr com.apple.quarantine "/Applications/Meego Lite.app"
   ```

#### Windows

1. 双击 `.exe` → 一路下一步
2. SmartScreen 提示"未识别的应用"时，点"更多信息" → "仍要运行"

> 💡 正式分发建议申请 Apple Developer ID（99 美元/年）+ Windows 代码签名证书，避免拦截提示。

### 7.4 Prisma 跨平台打包

需要支持多平台时，修改 `prisma/schema.prisma`：

```prisma
generator client {
  provider      = "prisma-client-js"
  binaryTargets = ["native", "darwin", "darwin-arm64", "windows", "linux-musl-openssl-3.0.x"]
}
```

然后重新生成：

```bash
bun run db:generate
```

## 八、方式 F：Vercel + Neon 云托管（海外/Serverless）

> 零运维，适合海外用户、个人 SaaS 项目。**国内访问较慢**，国内用户优先选方式 D。

### 8.1 整体架构

```
用户浏览器
    ↓ HTTPS
Vercel (Next.js)
    ↓ TCP
Neon (PostgreSQL)
```

**关键变化**：必须从 SQLite 切换到 PostgreSQL（Vercel Serverless 文件系统只读）。

### 8.2 准备数据库（Neon）

1. 注册 [https://neon.tech](https://neon.tech)
2. New Project → 选区域 `Asia Pacific - Singapore`
3. 复制连接字符串：
   ```
   postgresql://user:pass@ep-xxx.aws.neon.tech/neondb?sslmode=require
   ```

### 8.3 改 Schema

修改 `prisma/schema.prisma`：

```prisma
datasource db {
  provider = "postgresql"   // ← 从 sqlite 改成 postgresql
  url      = env("DATABASE_URL")
}
```

推送到云数据库：

```bash
export DATABASE_URL="postgresql://..."
bun run db:generate
bun run db:push
```

### 8.4 推到 GitHub + 部署 Vercel

```bash
git init
git add . && git commit -m "feat: cloud deploy"
git remote add origin git@github.com:你的账号/meego-lite.git
git push -u origin main
```

然后：

1. 登录 [https://vercel.com](https://vercel.com)
2. Add New → Project → 选仓库
3. Environment Variables 添加：
   - `DATABASE_URL` = Neon 连接串
   - `NEXTAUTH_SECRET` = 随机字符串
   - `NEXTAUTH_URL` = Vercel 给的域名
   - `AUTH_SECRET` = 随机字符串
   - SMTP 相关
4. Deploy

### 8.5 成本预估

| 用户量 | 方案 | 月成本 |
|--------|------|--------|
| 个人/几人 | Vercel + Neon 免费 | **¥0** |
| 10-30 人 | Vercel Pro + Neon 免费 | ~¥150 |
| 百人级 | Vercel Pro + Neon Launch | ¥350-700 |

## 九、数据库管理（重要）

### 9.1 项目采用"双轨迁移"策略

| 端 | 同步方式 | 触发时机 |
|----|---------|---------|
| **服务器/Docker** | `bunx prisma db push`（标准方式）| 每次发版手动 |
| **Electron 桌面 App** | `db-migrations.ts` 运行时自动迁移 | 启动后第一次 API 请求 |

> 服务器走标准 Prisma 流程，Electron 走运行时自动迁移，**用户零感知**。

### 9.2 Schema 改动后必做

每次修改 `prisma/schema.prisma`，**必须同时更新** `src/lib/db-migrations.ts`：

```typescript
// 1. schema.prisma 加字段
model User {
  newField String?
}

// 2. db-migrations.ts 加对应迁移函数（Electron 自动迁移用）
export async function ensureUserNewFieldColumn(client: RawSqlDb) {
  // 检查列是否存在，不存在则 ALTER TABLE ADD COLUMN
}

// 3. 在 ensureDatabaseSchema() 里调用
await ensureUserNewFieldColumn(db);
```

### 9.3 Prisma 命令对照表

| 命令 | 作用 | 风险 |
|------|------|-----|
| `bun run db:generate` | 生成 Prisma Client（代码层）| 0 |
| `bun run db:push` | 同步 schema 到数据库（保留数据）| 低 |
| `bun run db:reset` | ⚠️ **清空所有数据** | 高（仅开发用）|

**生产环境只用前两条**。

### 9.4 服务器发版（一键）

```bash
make update
# 内部自动: db-backup → git pull → prisma generate → prisma db push → build → restart
```

## 十、数据备份与恢复

### 10.1 各方式的数据位置

| 运行方式 | 数据库位置 |
|---------|----------|
| 方式 A | `项目目录/prisma/dev.db` |
| 方式 B | `项目目录/prisma/dev.db` |
| 方式 C | Docker 数据卷 `meego-lite_meego-db` |
| 方式 D | `/opt/Meego-Lite/prisma/prod.db` |
| 方式 E | `~/Library/Application Support/Meego Lite/meego-lite.db`（Mac）|
| 方式 F | Neon / Supabase 云数据库 |

### 10.2 备份命令

#### 本地（方式 A/B）

```bash
cp prisma/dev.db ~/Desktop/meego-backup-$(date +%F).db
```

#### Docker（方式 C）

```bash
docker cp meego-lite:/app/db/custom.db ~/Desktop/meego-backup-$(date +%F).db
```

#### 阿里云服务器（方式 D）

```bash
# 手动一次
make db-backup

# 自动每天 3 点备份（推荐）
crontab -e
# 加一行：
0 3 * * * cp /opt/Meego-Lite/prisma/prod.db /opt/backup/prod-$(date +\%Y\%m\%d).db && find /opt/backup -mtime +30 -delete
```

### 10.3 恢复命令

```bash
# 方式 A/B
cp ~/Desktop/meego-backup-XXXX.db prisma/dev.db

# 方式 C（先停容器）
docker compose stop
docker cp ~/Desktop/meego-backup-XXXX.db meego-lite:/app/db/custom.db
docker compose start

# 方式 D（服务器）
sudo cp /opt/backup/prod-XXXX.db /opt/Meego-Lite/prisma/prod.db
make restart
```

## 十一、常见问题

### Q1：`command not found: bun`

完全关闭终端重启，再运行 `bun --version`。还不行就重新执行 `curl -fsSL https://bun.sh/install | bash`。

### Q2：3000 端口被占用

```bash
lsof -i :3000     # 查看占用进程
# 换端口启动
bun run dev -- -p 3001
```

### Q3：报错 `DATABASE_URL` 找不到

项目根目录创建 `.env`：

```bash
cp .env.example .env
```

### Q4：换电脑使用

1. 整个项目目录拷过去
2. **`prisma/dev.db` 一并拷过去**（你的数据）
3. 新电脑执行 `bun install && bun run dev` 即可

### Q5：Docker 数据丢了

最常见原因：用了 `docker compose down -v`。**正确的停止方式**用 `docker compose stop` 或 `docker compose down`（不加 `-v`）。

### Q6：阿里云访问不了 80 端口

未备案。临时用 8080 端口（`make nginx-temp` 已配置好），需要在阿里云控制台防火墙开放 8080。

### Q7：阿里云 install 报 `ConnectionRefused`

切换国内镜像：

```bash
mkdir -p ~/.bun
echo '[install]
registry = "https://registry.npmmirror.com"' > ~/.bun/bunfig.toml

bun install --registry=https://registry.npmmirror.com
```

或用 `make install`，内置已切镜像。

### Q8：Electron 打包卡在下载 Electron

```bash
export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
bun run electron:build:mac
```

### Q9：改了 schema.prisma 后服务器报错

发版时跑了 `make update`（含 `prisma db push`）吗？或手动同步：

```bash
bunx prisma generate
bunx prisma db push
pm2 restart meego-lite
```

### Q10：邮箱验证码收不到

检查 `.env` 里的 SMTP 配置：

- `SMTP_PASS` 必须是邮箱**授权码**，不是登录密码
- 阿里云服务器**默认封禁 25 端口**，必须用 465 + `SMTP_SECURE=true`
- QQ 邮箱授权码需在邮箱设置 → 账户 → 开启 IMAP/SMTP 服务获取

## 总结：选你的部署方式

| 你的需求 | 推荐方式 | 一句话指引 |
|---------|---------|----------|
| 个人体验 | A | `bun install && bun run dev` |
| 个人长期工作台 | B | PM2 后台跑 |
| NAS / 内网共享 | C | `docker compose up -d` |
| 团队 / 公网服务 | **D ⭐** | `make first-deploy` |
| 桌面 App 分发 | E | `bun run electron:build:mac` |
| 海外 SaaS / 零运维 | F | Vercel + Neon |

**最佳实践**：

- 个人开发 → 方式 A
- 团队上线 → 方式 D（看 [`阿里云部署完整方案.md`](./阿里云部署完整方案.md)）
- 给老板/客户体验 → 方式 E（桌面 App）
