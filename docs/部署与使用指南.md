## 一、准备工作（只需做一次）

### 1. 打开"终端"

- **Mac 用户**：按下 `Command + 空格`，输入 `终端`，回车打开。
- 后续所有以 `$` 开头的命令，都是在这个"终端"窗口里输入的（输入时 **不要** 包含 `$` 符号）。

### 2. 进入项目目录

把下面这行命令复制进终端，回车：

```bash
cd ./your/project/meego-lite
```

> 💡 如果你的项目放在别的位置，把上面路径换成你自己的项目路径。

## 二、运行方式选择

| 方式 | 适合谁 | 优点 | 缺点 |
|------|--------|------|------|
| **A. 本地直接运行** | 个人使用 ✅ | 启动快、占资源少、数据库文件随手可见 | 第一次需要装 Bun |
| **B. Docker 运行** | 想部署到服务器 / 团队共享 | 环境隔离、迁移方便 | 需要安装 Docker、占用资源多 |
| **C. 打包成桌面 App** | 给同事/客户分发使用 ✅ | 双击即用、跨平台（Mac/Win）、用户无需装环境 | 打包流程稍复杂、安装包体积大（~150MB） |
| **D. 部署到云端** | 团队多人共享数据 ✅ | 零安装、自动 HTTPS、跨设备访问、数据共享 | 需要换数据库（SQLite → Postgres）、有少量云成本 |

**👉 个人本地使用推荐 [方式 A](#三方式-a本地直接运行推荐个人使用)。**
**👉 给非技术用户分发推荐 [方式 C](#五方式-c打包成桌面-appmacwindows-双平台分发)。**
**👉 团队多人协作推荐 [方式 D](#六方式-d部署到云端团队多人共享最省心)。**

## 三、方式 A：本地直接运行（推荐个人使用）

### 步骤 1：安装 Bun（一种 JavaScript 运行环境）

在终端粘贴下面这行命令并回车：

```bash
curl -fsSL https://bun.sh/install | bash
```

安装完后，**关闭并重新打开终端**让设置生效。

验证是否安装成功：

```bash
bun --version
```

如果输出了一串版本号（例如 `1.1.x`），说明安装成功。

---

### 步骤 2：进入项目目录

```bash
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite
```

---

### 步骤 3：安装项目依赖（第一次需要，可能需要等几分钟）

```bash
bun install
```

看到类似 `Done` 或 `installed` 的提示就表示完成。

---

### 步骤 4：初始化数据库（第一次需要）

按顺序执行下面**两条**命令：

```bash
bun run db:generate
```

```bash
bun run db:push
```

完成后，会在项目的 `prisma/` 目录下生成一个 `dev.db` 文件 —— 这就是你的数据库。

---

### 步骤 5：启动应用

```bash
bun run dev
```

看到类似下面的输出，就表示启动成功：

```
- Local:   http://localhost:3000
```

**用浏览器打开 http://localhost:3000** ，就可以使用了。

> ⚠️ **不要关闭终端窗口**，关闭终端会让应用停止。

---

### 步骤 6：以后每次启动

之前装过 Bun 和依赖了，以后只需两步：

```bash
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite
```

```bash
bun run dev
```

打开 http://localhost:3000 即可。

---

### 步骤 7：停止应用

在运行应用的终端窗口里按 **`Control + C`** 即可停止。

去官网下载安装：https://www.docker.com/products/docker-desktop/

安装完打开 Docker Desktop，等到左下角显示绿色状态（运行中）。

### 步骤 2：进入项目目录

```bash
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite
```

---

### 步骤 3：一键启动

```bash
docker compose up -d --build
```

第一次会下载镜像 + 构建，需要等几分钟。完成后访问 http://localhost:3000 即可使用。

---

### 步骤 4：常用 Docker 命令

| 命令 | 作用 |
|------|------|
| `docker compose ps` | 查看是否在运行 |
| `docker compose logs -f` | 查看运行日志（按 `Control + C` 退出查看） |
| `docker compose stop` | 暂停服务（数据保留） |
| `docker compose start` | 重新启动 |
| `docker compose down` | 停止并清理容器（**数据保留** ✅） |
| `docker compose down -v` | 停止并 **删除数据卷**（⚠️ **数据会丢失！慎用**） |

## 五、方式 C：打包成桌面 App（Mac/Windows 双平台分发）

### 适用场景

如果你想把 Meego Lite **分发给同事或客户**，又不想让他们装 Bun、Node、终端命令一通操作，最好的方式就是打包成一个真正的桌面应用：

- **Mac 用户**：拿到一个 `.dmg` 文件，双击安装到"应用程序"，跟普通软件一样使用
- **Windows 用户**：拿到一个 `.exe` 安装包，下一步下一步装完即用

底层使用 [Electron](https://www.electronjs.org/) 框架，把 Chromium 浏览器内核 + Node.js 运行时打包到一个可执行文件里，用户**完全不用关心环境**。

---

### 原理简介（了解即可）

```
双击 App
  ↓
Electron 主进程启动（electron/main.js）
  ↓
首次启动：把 prisma/dev.db 复制到用户数据目录
  ↓
fork 一个 Node 子进程跑 Next.js standalone server（127.0.0.1:3000）
  ↓
创建窗口加载 http://127.0.0.1:3000
  ↓
用户看到一个原生桌面窗口，里面跑的就是你的 Web 应用
```

**数据存储位置**（每台电脑独立一份本地 SQLite 数据库）：

| 系统 | 数据库路径 |
|------|----------|
| macOS | `~/Library/Application Support/Meego Lite/app.db` |
| Windows | `C:\Users\<用户名>\AppData\Roaming\Meego Lite\app.db` |
| Linux | `~/.config/Meego Lite/app.db` |

> ⚠️ 因为是每台电脑独立的本地库，**多人使用时数据不互通**。需要协作请改用方式 B 部署到服务器。

---

### 步骤 1：安装 Electron 依赖

```bash
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite
bun add -D electron electron-builder
```

> 项目里已经预置好了 `electron/main.js` 主进程文件以及 `package.json` 中的打包配置，可以直接用。

---

### 步骤 2：本地预览（开发模式）

打包之前先验证能不能正常跑起来：

```bash
bun run electron:dev
```

这条命令会：
1. 自动执行 `next build` 生成 standalone 产物
2. 启动 Electron 窗口，加载本地的 Next.js 服务

如果窗口正常打开并显示应用界面，就说明集成成功。

---

### 步骤 3：打包成安装包

#### Mac 平台（在 Mac 电脑上执行）

```bash
bun run electron:build:mac
```

完成后产物在 `dist-electron/` 目录下，通常包含：

- `Meego Lite-0.2.0-arm64.dmg`（Apple Silicon 芯片）
- `Meego Lite-0.2.0.dmg`（Intel 芯片）
- `Meego Lite-0.2.0-mac.zip`

把 `.dmg` 发给 Mac 用户就行。

> ⚠️ **国内网络打包失败（`ETIMEDOUT 20.205.243.166:443`）解决方案**
>
> electron-builder 在打包阶段需要从 GitHub 下载 Electron 二进制和 `electron-builder-binaries` 等资源，国内直连 GitHub 经常超时。**打包前先设置以下两个环境变量切换到国内镜像即可**：
>
> ```bash
> export ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/
> export ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
> bun run electron:build:mac
> ```
>
> 说明：
> - `ELECTRON_MIRROR`：Electron 运行时下载源
> - `ELECTRON_BUILDER_BINARIES_MIRROR`：electron-builder 所需辅助二进制（如 `dmg-license`、`7zip-bin` 等）的下载源
> - 这两行只对当前终端窗口生效，关掉终端就失效。需要长期生效可写入 `~/.zshrc`。
>
> 若仍失败，可使用 `bun run build && npx electron-builder --mac --dir` 跳过 DMG 打包，只生成 `.app` 文件用于本地验证。

#### Windows 平台（在 Windows 电脑上执行）

```bash
bun run electron:build:win
```

产物：`dist-electron/Meego Lite Setup 0.2.0.exe`，发给 Windows 用户双击即可安装。

> 💡 **跨平台打包限制**：Mac 上不能打 Windows 包，反之亦然。建议用 GitHub Actions CI 同时跑两个平台的打包任务。

#### Mac 用户

1. 双击 `.dmg` 文件
2. 把 Meego Lite 图标拖入"应用程序"
3. 在 Launchpad 找到 Meego Lite 双击打开

- **macOS 15+ (Sequoia)**：去"系统设置 → 隐私与安全性"，最下方会有提示，点"仍要打开"

#### Windows 用户

1. 双击 `.exe` 安装包
2. 一路下一步，安装完成
3. 桌面/开始菜单找到 Meego Lite 双击打开

⚠️ Windows 未签名的 exe 可能被 Defender 提示"未识别的应用"，点"更多信息" → "仍要运行"即可。

### 步骤 5：处理 Prisma 跨平台问题（可选）

Prisma 的 query engine 是平台相关的二进制文件。**如果想在 Mac 上同时打 Windows 包**，需要修改 `prisma/schema.prisma`：

```prisma
generator client {
  provider      = "prisma-client-js"
  binaryTargets = ["native", "darwin", "darwin-arm64", "windows", "linux-musl-openssl-3.0.x"]
}
```

然后重新生成：

```bash
bun run db:generate
```

这样 `node_modules/.prisma/client/` 目录下会同时有多个平台的 query engine 二进制，打包时都会带上。

### 常见疑问

**Q：为什么 Web App 能在桌面运行？**
Electron 内置了 Chromium（精简版浏览器）+ Node.js（服务端运行时），相当于把"浏览器 + 服务器"打包进了一个可执行文件里。用户看到的"桌面窗口"本质上是一个砍掉地址栏的 Chrome，加载的是本地启动的 `http://127.0.0.1:3000`。

**Q：包体积大概多大？**
Mac dmg 约 90-150MB，Windows exe 约 80-130MB（主要是 Chromium 内核占的）。这是 Electron 应用的常态，VSCode、Slack、Notion 都差不多。

**Q：用户的数据存在哪？换电脑怎么办？**
存在用户系统的 `userData` 目录（见上面的表格）。换电脑只要把 `app.db` 文件拷到新电脑对应路径就行。

**Q：要不要做代码签名？**
不是必须的。**给少数熟人/同事用**：可以不签名，附上"右键打开"的截图说明就行。**公开商业分发**：建议买苹果开发者账号（99 美元/年）做签名 + 公证，体验最好。

**Q：能不能自动更新？**
可以，`electron-builder` 支持配合 `electron-updater` 实现自动更新（需要把更新包传到 GitHub Release / 私有 OSS 等）。当前最小版未启用，需要再扩展配置。

## 六、方式 D：部署到云端（团队多人共享，最省心）

### 适用场景

- **团队协作**：多人需要看到同一份任务和数据
- **跨设备使用**：自己想在公司电脑、家里电脑、手机上都能访问
- **零分发成本**：不用给每个用户打包安装，发个网址就行
- **始终最新版**：你更新一次，所有人立刻用上新功能

> 💡 这也是绝大多数 SaaS 软件的部署模式（飞书、Notion、Linear 等都是云端部署）。

---

### 整体架构

```
┌─────────────────────────┐
│  用户的浏览器/手机       │
└────────────┬────────────┘
             │ HTTPS
             ▼
┌─────────────────────────┐
│  Vercel / Railway 等     │← 跑你的 Next.js 应用
│  （应用托管平台）         │
└────────────┬────────────┘
             │ TCP
             ▼
┌─────────────────────────┐
│  Neon / Supabase 等      │← 跑 PostgreSQL 数据库
│  （云数据库）             │
└─────────────────────────┘
```

**关键变化**：本地的 SQLite 文件需要换成云上的 PostgreSQL（或 MySQL），因为：
1. SQLite 是单文件本地数据库，云函数/容器重启时会丢
2. SQLite 不支持高并发，多人同时操作会卡
3. Vercel 这类 Serverless 平台文件系统是只读的，根本不让写

---

### 推荐组合（按上手难度排序）

| 应用托管 | 数据库 | 免费额度 | 适合场景 |
|---------|-------|---------|---------|
| **Vercel** + **Neon** | Postgres | 应用 100GB 流量/月、Neon 0.5GB 存储 | Next.js 官方组合，最丝滑 ✅ |
| **Railway** | 内置 Postgres | 5 美元/月免费额度 | 一站式，不想分开管理 |
| **Cloudflare Pages** + **Supabase** | Postgres | 无限请求 + 500MB 数据库 | 国内访问快，免费额度大方 |
| **自己的服务器**（阿里云/腾讯云） | 自建 Postgres | 看你买的配置 | 数据私有、可控 |

下面以**最推荐的 Vercel + Neon 组合**为例详细讲解。

### 步骤 1：准备数据库（Neon）

1. 访问 [https://neon.tech](https://neon.tech)，用 GitHub 账号登录
2. 点 "New Project"，选择最近的区域（建议 `Asia Pacific - Singapore`）
3. 创建后会得到一个连接字符串，长这样：

```
postgresql://username:password@ep-xxx.aws.neon.tech/neondb?sslmode=require
```

把它**复制保存**，下一步要用。

---

### 步骤 2：改造项目支持 Postgres

#### 2.1 修改 `prisma/schema.prisma`

```prisma
datasource db {
  provider = "postgresql"   // ← 从 sqlite 改成 postgresql
  url      = env("DATABASE_URL")
}
```

#### 2.2 调整 schema 兼容 Postgres（重要）

SQLite 没有数组类型，所以原来用了 `String` 存 JSON。Postgres 支持原生数组，可以选择性优化（也可以保持 String 不变省事）：

```prisma
model Task {
  // 原来：tags String?
  // 可选优化为：tags String[] @default([])
  tags String?  // 保持不变也能跑
}
```

#### 2.3 推送 schema 到云数据库

在本地终端跑：

```bash
# 临时设置数据库连接（替换成你 Neon 的连接串）
export DATABASE_URL="postgresql://username:password@ep-xxx.aws.neon.tech/neondb?sslmode=require"

# 重新生成 Prisma client
bun run db:generate

# 把表结构同步到云数据库
bun run db:push
```

完成后在 Neon 控制台的 "Tables" 标签下能看到 `User`、`Task`、`Comment`、`Notification` 四张表。

---

### 步骤 3：把代码推到 GitHub

Vercel 是通过 GitHub 仓库自动部署的，所以先把代码推上去：

```bash
cd /Users/bytedance/Desktop/KOK/沉淀/APP/meego-lite

# 如果还没初始化 git
git init
git add .
git commit -m "feat: ready for cloud deploy"

# 关联到 GitHub 仓库（先在 github.com 建个空仓库）
git remote add origin git@github.com:你的用户名/meego-lite.git
git push -u origin main
```

---

### 步骤 4：在 Vercel 部署

1. 访问 [https://vercel.com](https://vercel.com)，用 GitHub 登录
2. 点 "Add New" → "Project" → 选刚刚推上去的 `meego-lite` 仓库
3. **Framework Preset** 会自动识别为 Next.js，无需修改
4. 展开 **Environment Variables** 部分，添加：

| Name | Value |
|------|-------|
| `DATABASE_URL` | 你 Neon 的连接字符串 |
| `NEXTAUTH_SECRET` | 任意随机字符串（如 `openssl rand -base64 32` 生成） |
| `NEXTAUTH_URL` | 部署成功后的域名（先随便填，部署完再回来改） |

5. 点 **Deploy**，等 2-5 分钟

部署成功后，会得到一个网址，例如：

```
https://meego-lite-xxx.vercel.app
```

打开就能用了，**任何人凭这个网址都能访问**。

---

### 步骤 5：绑定自定义域名（可选）

默认的 `*.vercel.app` 域名国内访问可能慢，绑个自己的域名更稳定：

1. 在 Vercel 项目 → **Settings** → **Domains**
2. 输入你买的域名（如 `meego.yourdomain.com`）
3. 按提示去你的域名 DNS 设置里加一条 `CNAME` 记录指向 `cname.vercel-dns.com`
4. Vercel 会自动签发 HTTPS 证书

### 步骤 6：以后怎么更新？

**核心优势**：本地改完代码 → `git push` → Vercel 自动重新构建 + 上线。

```bash
# 本地修完代码
git add .
git commit -m "fix: 修复任务列表 bug"
git push

# Vercel 自动检测到 push，开始重新部署
# 等 2 分钟后，所有用户访问的就是新版本了
```

### 国内访问优化

**问题**：Vercel 在国内有时会慢甚至打不开。

**解决方案**：

1. **CDN 加速**：用 Cloudflare 在前面套一层（免费）
2. **改用 Cloudflare Pages**：原生 Cloudflare 服务，国内访问明显更快
3. **国内云**：阿里云函数计算 / 腾讯云 SCF / 华为云函数 + 自建 Postgres，速度最快但配置最复杂
4. **走 Docker 部署到国内 VPS**：直接用本指南的 [方式 B](#四方式-bdocker-运行适合服务器部署)，把 docker-compose 部署到阿里云 ECS / 腾讯云轻量服务器（最低 60 元/月）

**Q：Vercel 免费版有什么限制？**

**Q：登录鉴权怎么办？**

- `cloud`：用 Postgres，给云端用

### 数据存储位置

| 运行方式 | 数据库文件位置 | 关闭后数据是否保留 |
|---------|--------------|------------------|
| 方式 A（本地） | `项目目录/prisma/dev.db` | ✅ 保留 |
| 方式 B（Docker） | Docker 数据卷 `meego-lite_meego-db` 中的 `/app/db/custom.db` | ✅ 保留（除非用了 `down -v`） |

| 方式 D（云端） | Neon / Supabase 等云数据库（PostgreSQL） | ✅ 由云厂商负责持久化 |

**结论：正常关闭、重启电脑都不会丢数据。**

## 九、常见问题

### Q1：终端提示 `command not found: bun`

**原因**：Bun 没装好，或终端没重启。

**解决**：完全关闭终端（`Command + Q`），再重新打开终端，重新执行 `bun --version` 试试。如果还不行，重新执行步骤 1 的安装命令。

---

### Q2：访问 http://localhost:3000 打不开

**原因**：应用没启动起来，或者端口被占用。

**解决**：
1. 确认运行 `bun run dev` 的终端没报红色错误
2. 检查 3000 端口是否被占用：
   ```bash
   lsof -i :3000
   ```
   如果有进程占用，关掉那个进程，或者换个端口启动：
   ```bash
   bun run dev -- -p 3001
   ```
   然后访问 http://localhost:3001

---

### Q3：报错 `DATABASE_URL` 找不到

**原因**：项目根目录缺少 `.env` 文件。

**解决**：在项目根目录创建 `.env` 文件，写入一行：

```
DATABASE_URL="file:./dev.db"
```

---

### Q4：我想换电脑使用怎么办？

1. 把整个项目目录拷到新电脑
2. 把 `prisma/dev.db` 数据库文件一起拷过去（**这是你的数据**）
3. 在新电脑上执行：
   ```bash
   bun install
   bun run dev
   ```
   不需要重新执行 `db:push`，原来的数据就直接能用了。

---

### Q5：Docker 运行时数据丢了？

**最常见原因**：用了 `docker compose down -v`（带 `-v` 会删数据卷）。

**正确停止方式**：用 `docker compose stop` 或 `docker compose down`（**不要加 `-v`**）。

---

### Q6：想本地运行 + Docker 运行切换，数据能共用吗？

**默认不共用**：
- 本地运行用的是 `项目目录/prisma/dev.db`
- Docker 运行用的是 Docker 数据卷里的 `custom.db`

如果想共用，需要修改 `docker-compose.yml`，把数据卷挂载改成绑定到本地 `prisma/` 目录。**不建议混用**，容易乱。建议**只选一种方式**长期使用。
