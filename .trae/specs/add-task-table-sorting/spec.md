# 任务列表表格排序 Spec

## Why
当前任务列表（列表视图）只能按后端返回的默认顺序展示，用户无法按关键列重新排序，难以快速定位关心的任务。需要支持点击列头按「项目」「状态」「优先级」「截止时间」「负责人」排序。

## What Changes
- 任务列表表格的 5 个列头（项目、状态、优先级、截止时间、负责人）支持点击排序，并显示升/降序指示图标。
- 默认按「项目」升序排序。
- 状态列排序使用业务自定义顺序：待开始 → 进行中 → 已完成 → 已关闭 → 已暂停（区别于现有的状态流转顺序 `TASK_STATUS_ORDER`）。
- 排序为纯前端实现，不改动后端接口与数据请求。

## ADDED Requirements

### Requirement: 列头排序交互
任务列表（列表视图）表格的「项目」「状态」「优先级」「截止时间」「负责人」列头 SHALL 支持点击排序，并通过图标指示当前排序列与方向（升序/降序）。

#### Scenario: 点击列头排序
- **WHEN** 用户点击某个可排序列头
- **THEN** 表格按该列升序重新排列，并在该列头显示升序图标

#### Scenario: 再次点击切换方向
- **WHEN** 用户再次点击当前排序列的列头
- **THEN** 表格切换为降序排列，并在该列头显示降序图标

#### Scenario: 切换排序列
- **WHEN** 用户点击另一个可排序列头
- **THEN** 表格改为按新列升序排列，原列头不再显示排序图标

### Requirement: 默认排序
任务列表（列表视图）首次加载时 SHALL 默认按「项目」升序排序。

#### Scenario: 首次进入列表
- **WHEN** 用户进入任务列表视图且未手动点击任何列头
- **THEN** 任务按项目名称升序排列，无关联项目（暂不关联）的任务排在最后

### Requirement: 状态列业务排序
按「状态」排序时 SHALL 使用业务自定义顺序：待开始(todo) → 进行中(in_progress) → 已完成(done) → 已关闭(closed) → 已暂停(paused)。

#### Scenario: 升序按状态排序
- **WHEN** 用户按状态升序排序
- **THEN** 任务依次按 待开始、进行中、已完成、已关闭、已暂停 分组排列

### Requirement: 各列排序规则
排序 SHALL 按列的语义进行：
- 项目：按项目名称本地化字符串排序，无项目的排在最后。
- 状态：按状态业务顺序排序。
- 优先级：按 P0 → P1 → P2 → P3 顺序排序。
- 截止时间：按截止日期先后排序，「长期任务」（无截止时间）排在最后。
- 负责人：按责任人姓名本地化字符串排序，未分配的排在最后。

#### Scenario: 截止时间排序中的长期任务
- **WHEN** 用户按截止时间升序排序
- **THEN** 有截止时间的任务按日期由近到远排列，长期任务（无截止时间）统一排在最后
